Process Structures with ChemScript in Registration System

 

As a major feature of Registration System 10.0, the ChemScript feature is functioning as follows per our discussion:

  • Provide structure transformation in Reg database

From Registration System, after a structure is inserted into temporary table, Reg calls up to ChemScript via webservice, to perform transformation of the structure. The ChemScript scripts are defined by Reg admin at certain location, which is defined as a setting in reg ini file.

  • Provide built-in Salt Stripping based on IMMETECH requests.

The built-in Salt Stripping happens when the structure is inserted into the permanent table. Reg calls out the ChemScript web service with the structure and other info, the web service returns the modified molecule and mutiple fragments, being salts and solvates. Then Reg goes on the logic and make decisions of processing the salts and solvates.

Possible technical routes:

  • Reg asp --> PL/SQL --> ChemScript web service --> ChemScript --> PL/SQL finish up the reg logic --> return to asp

  • Reg asp --> ChemScript web service --> ChemScript -->return to asp --> asp finish up the reg logic

Tasks:

  • Create GUI for manage and maintain Fragments table.

Fragments table is where the fragments like salts and solvates are defined, it is focusing on structure of the fragments, where SALTS and SOLVANTS table are focusing on properties and other info.

  • Web Service location configured in the ini file.

  • ChemScript file location is configured in teh ini file.

  • ini setting to turn salt stripping on and off.

Time Line:

  • Deliver mid summer.
