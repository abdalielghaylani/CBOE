Creating a Dataview from the Master

To create a dataview from the Master dataview:

  1. On the ChemBioOffice Enterprise Manager main page, click on "Create from master" icon. The DataView Manager window opens.

  1. In the Name and Description text boxes, enter a name for the new dataview and a brief description.

  1. In the dropdown lists, select a data source and a schema and then specify a base table by selecting its corresponding radio button.
    Note:If you select a base table that does not have a primary key the following message displays:

  2. After completing the selections, click Ok. The following window appears:

  1. To add further tables to the base table, click Add Table at the top of the window. The following window appears:
    Note: The names of tables that have already been added appear in Red.

  1. To add a table, click on its corresponding Add button. The selected tables get added under Tables to add column at the right frame.

  1. A filtering field “Filter table name” is available. Inside the filtering field, start typing the name of the table to add, this will reduce the number of options and will help you extracting the desired table from large lists.
  2. It is possible as well, to add table from different data source/schema. Select the desired data source/schema from the Data source/Schema dropdown lists to display its corresponding tables and add the desired tables.
  3. After completing the table’s addition, click Save at the bottom of the window.

Note: Make sure to select the Primary Key in the base table while creating relations between Base and Child tables.

Editing Dataviews

Name and Description

It is possible to edit the name and description of the dataview by clicking Name and Description link and entering new data.

Security Settings

It is possible to apply security settings for the dataview. Two options are available. A dataview may be:

  • Public (Seen by all users and roles)
  • Not public (Seen by specific users and roles)

If “I want to choose the users and roles for this Data View” radio button is selected, the following window appears, showing the available users and roles and allowing the dataview owner to give access permissions to specific users and roles:

Select the Base Table

Note:You can change the base table in a dataview prior to submitting the data view. Once a dataview is submitted you cannot change the base table.

To change the base table:

  • Click “Select Base Table” , the following dialog appears showing the list of available tables

  • Click the radio button beside the table to make it the base table and click Ok.

Remove all tables from a Schema

To remove all tables from a schema, click Remove.

Working with Tables

Editing tables

After you add tables to the dataview, you can edit each table as needed.

  1. From the list of tables, click on the name of the table that you want to edit, its corresponding Table Summary pane appears.
  2. Click Edit Table.

  1. The Edit Table and Fields pane appears.

Within this table the user is allowed to perform the following edits:

SELECTING THE PRIMARY KEY

If the table that you are editing does not have a primary key, you can identify the primary key from the Edit Table and Fields window by selecting the corresponding radio button under the Primary Key column.

Note: you can select any field except of data type DATE/TIME/CLOB/BLOB as primary key however you will get a warning message as shown in the screen shot below:

EDITING THE TABLE ALIAS

The Alias field is editable; modify the Alias name if desired.

CREATING A FIELD ALIAS

  1. In the list of fields, select the Name for the field that you want to create an alias for.
  2. Click Create Field Alias.

  1. A new field appears in the field list with the _Alias extension.

CHOOSING LOOKUP FIELD

  1. In the LookUp column, click Choose corresponding to the field that you want to assign as the lookup. The Edit a lookup pane appears.
    Note: For the existing lookup field in the Master Dataview to be visible, use Add Tables link to select the table with the defined lookup in the Master Dataview. You can then edit it by clicking on its corresponding link under the LookUp column.
  2. In the left column, expand the tree and select the table to use.
  3. In the right column, select the field in the Select the display field list and a Sort Direction from the dropdown list.
  4. Click Submit.

 

ASSIGNING A PARENT COLUMN

  1. Under Parent Column, click Choose for the field you want to assign as the parent. The Define Relationships pane appears.
  2. Select a Parent Schema, a parent table, a joining field and then the Join type.
  3. Once the selection are completed, click Submit.

ASSIGNING AN INDEX TYPE

In the Index Type column, select an option from the dropdown list.

ASSIGNING A CONTENT TYPE

In the Content Type column, select an option from the dropdown list.

Note: If you want to show the structure field in the Dataview, then you have to configure the "Structure Aggregate" as shown below:

REMOVING FIELDS

To remove a field from the table, click Remove in the Action column.

QUERY FIELDS

You can use the Query field to specify fields that would be automatically generated on the Query tab of other ChemBioOffice Enterprise applications such as Datalytix. This field also lets you specify if you want to index the field you have selected. Indexing a field enables quick data searching in Datalytix. The fields that are indexed have a Green background and those that are not indexed have Red background as shown below:

Dataview Manager needs to be configured to enable or disable the prompt to index fields. To enable the prompt to index fields:

  1. In the DataView Manager home page, click Manage Configuration icon.
  2. Under DataViewManager settings tab, set "Apply _Indexing'="Enable".
  3. Click Save.

When you select a field which is not indexed, a prompt appears as shown below. Click Yes to set an index on the particular field.

Note: Indexing functionality will not be available if you select a 'view' instead of a table even if “Apply_Indexing” function is enabled.

Note:Once an index is created on a field, you cannot drop it from the application. It can be dropped only from the database.

DISPLAY FIELDS

You can define default fields in the metadata of a dataview so that this information can be used when defining the data set in Spotfire. To make a particular field default, check its corresponding checkbox under Display column.

SAVING YOUR CHANGES

After you finished editing the table, click OK in the Edit Table and Fields pane.

You can see the summary of relationships between tables when you select any of those tables.

Duplicating Tables

It is possible for users to duplicate existing tables. For more details please refer to Duplicating a Table topic.

Editing Tags

Dataview Manager allows user to define tagging categories on tables in the metadata of a Dataview so this information can be used when searching for tables while defining the data set in Spotfire. A particular table may have more than one tag.

To add tags, please follow the steps below:

  1. Click on Edit Tags link corresponding to a specific table, the following window appears:

  1. Enter the name of the tag and click Add.

  1. Add as much tags as you want and then click on Save.
  2. To delete a Tag, select it and click on Delete button.
  3. The tags get added within the Table Summary section

Reviewing the Validation Summary

When you finish selecting the dataview options, review the schema summary.

  1. After you review your schema summary, click Submit. The Validation Summary pane opens.

Note: The validation summary displays any errors that your dataview may contain. All displayed errors must be resolved before the dataview is created. To resolve an error, click its corresponding Resolve link.

  1. After you review the validation summary, click Submit. The Validation Summary pane closes.
  2. Click Home to return to the ChemBioOffice Manager.
