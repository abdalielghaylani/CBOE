User Guide

Defining Query Critiera

The available operators for each field are dependent on the data type for the selected field.
If a value is not defined for a field on the query form, it will not be used as part of the query when the Search button is clicked. If no query criteria are defined, clicking on the Search button performs a ‘retrieve all’ based on the data defined under the Result Definition tab.
If an invalid value is entered into the query form, based on field’s data type, a red exclamation point will appear to the right of the field when the Search button is clicked. The query will not proceed until the issue is resolved.
 

 
Note: If the field is a lookup field, Datalytix will query on the value of the field it is looking up, and not the original field value.

Filtering Child Hits based on Query

While searching a field in a child table, it is possible to show only child data matching entered criteria instead of pulling its parent record in the base table which will in turn retrieve all its child records. By enabling the ‘Filter child hits based on query’ checkbox, this option allows users to only retrieve the parent and child records that meet the search criteria.

Clearing the Query Form

To clear the query form criteria, click on the Clear form link. You will be prompted to confirm the clear query operation. Clearing the form restores the default state of the tab when the dataview was initially selected.
