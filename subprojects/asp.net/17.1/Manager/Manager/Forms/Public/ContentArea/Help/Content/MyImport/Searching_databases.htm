Searching databases

ChemBioOffice includes the ChemACX and drug databases and a sample database. In this guide we explain how to search the sample database. To search the other databases, refer to the ChemACX or Drug Degradation user guides (in the documentation library) or online help.
To select the sample database:
1. In the ChemBioOffice Enterprise Manager, click Search Databases under Chemical Databases. The database list opens.
2. In the database list, select Sample.The query form opens.
You can now search the Sample database. See "Basic searching".

Basic searching

To help you get started, we provide basic search instructions below. You can search using text, drawn structures, or a combination of both. Only those records that meet all your search criteria are returned.
To perform a simple search:
1. Enter your search criteria in the form fields.
2. (Optional) To perform a structure search:
a. draw the query structure in the ChemDraw plugin window.
b. Select the structure SearchType in the drop-down list.
3. Click Search. The results of the search appear in the Results List View page.
The Total Searchable Records number specifies how many records are being searched in the database.

 

After the search is complete you can examine the results in either List View or Detail View (See "Viewing search results").
You can also use wildcard characters when entering your search criteria. For information, see these topics:
"Structure searching"
"Formula searching"
"Numerical searching"

Clearing the search form

To delete the search criteria in all search fields, click Clear Form.

Retrieving all records

To retrieve all records in the database, click Retrieve All.

Viewing search results

You can display search results in either list view or detail view. The list view displays several records on each results page while the details view shows one record but includes more information.

 

The List view displays multiple records at a time.

To display the details for a record, click Show Details in list view.

To display the list of records, click Return to List in details view.

Navigating through records

Typically, not all search results will appear on one page. To navigate through pages click the navigation buttons at the top of the Results List View.

 

To go to a specific record, clickRec# and enter the record number in the dialog box. Click OK to display the record.
Add Mode
Switches to Add mode, which lets you add records to the database.

Text searching

In text searching, information can be entered using wildcard ("*") characters to broaden the search results or "=" to narrow to exact wording.
Note: You cannot enter "AND", "OR", or "NOT" in the text fields.
This table lists examples of text queries:

 

Text Entered

Results Found

ben

Retrieves records that start with "ben". For example, the search returns benzene and benzofuran, but not bromobenzene.

*ben

Retrieves records that have any character before "ben" and any character after "ben" in the name field. For example, the search returns benzene, bromobenzene, and benzofuran.

=Benzene

Retrieves the exact text typed. For example, the search returns Benzene.

Ben*chl

Retrieves compounds with "ben" at the beginning, any character after "ben" and before "chl" and any character after "chl". For example, the search returns Benzyl Chloride.

Structure searching

You can enter chemical structures as search parameters using the ChemDraw plug-in. Draw a structure in the plug-in window and ChemBioOffice will use the structure like any other query parameter to search the database. For information on drawing query structures, see the Chem & Bio Draw User Guide.
To perform a structure search:
1. If the list or form view currently appears, click New Query to open the Query Input Form.
2. In the Query Input Form, draw a chemical structure using the ChemDraw plug-in.
3. In the SearchType drop-down list, select one of the structure search options (see below).
4. Click Search.
ChemBioOffice Enterprise provides four structure search options:
Substructure
Full Structure
Exact Structure
Tanimoto Similarity
These options are describe below.

Substructure

Substructure search is the default structure search type. Substructure searching returns structures that meet these criteria:
Match the basic chemical structure specified in the query form.
Contain additional attachments at the open bond positions.
To perform a narrower or broader search, you can attach different features, such as atom lists and variable bond types to a structure.

 

Two examples of query structures. A) An element list search; B) A search for R-groups connected by a single or double bond.

For more information about query features and their effects, see the Chem & Bio Draw User Guide.

Full structure

Full structure searches retrieve structures that match the query structure. Isotopes are recognized as hits but structures with additional attachments are not.

Exact structure

Exact searching retrieves only structures that exactly match the structure you draw in the query input form. When matching structures, exact searching takes stereochemistry into account but does not recognize isotopes as hits.

Similarity (Tanimoto)

Tanimoto Similarity searching retrieves chemicals with structural features that correspond to those specified in the query input form. Similarity searches are by nature, "fuzzy". Tanimoto Similarity searches are useful if you have a general idea of the types of compounds you want, but do not know the precise target compound.
Tanimoto Similarity searching matches general structural features and not specific atoms or bonds. Therefore, you need not specify particular bonds when searching compounds based on Tanimoto similarity.
ChemBioOffice Enterprise uses a Tanimoto calculation to determine whether compounds are similar. You can specify the degree of similarity in the Preferences dialog box. For more information on setting preferences, see "Setting search preferences".

Formula searching

You can search by chemical formula using the syntax described in the examples shown below:

 

Formula Entered

Results Found

C6H6

Compounds with 6 carbon and 6 hydrogen atoms, plus any number of other elements. Finds C6H6, C6H6N2O.

=C6H6

Compounds with exactly 6 carbon atoms, 6 hydrogen atoms, and no other elements.

C6

Compounds with 6 carbon atoms plus any number of other elements.

C6 N1-3

Compounds with 6 carbon atoms, one to three nitrogen atoms, plus any number of other elements.
These rules apply in formula searches:
Symbols may be one or two letters. Symbols may be in upper or lower case. Ambiguities are resolved according to a set of rules. To avoid ambiguities:
Capitalize the symbols properly.
Insert spaces between symbols.
Element counts are single integers or ranges between two integers separated by a hyphen. If a count is omitted, it is assumed to be 1.
If the formula query contains no wild card character, such as '*', it is assumed to be an exact search. The search requires an exact formula match, containing no elements except those indicated. If an asterisk (*) is anywhere in the string, the search is a partial match and other elements may be present in the hits. Symbols may be repeated. For example, CH3CH3 is interpreted as C2H6.
Note: Formula searches are non-structural: CH3CH2OH matches dimethyl ether and ethyl alcohol because each compound has the formula: C2H6O.
Parentheses may be used to group elements and apply a count to the entire group. For example, (CH2)3 is interpreted as C3H6.
Spaces or non-alphanumeric characters except parentheses are ignored.
If a symbol is properly capitalized (first letter upper case, followed by zero, one, or two lower-case letters), then the longest valid symbol that matches is preferred. For example, "Phe" matches phenylalanine rather than phosphorus; "Co" matches cobalt.
If symbols are not properly capitalized then the shortest symbol that matches is preferred. For example, 'co' is interpreted as carbon-oxygen instead of cobalt. The exception is that if two characters represent a valid two-letter symbol and also a valid one-letter symbol followed by an invalid one, then the two-letter symbol is favored. cl is interpreted as chlorine and not as carbon followed by the invalid symbol L.

Numerical searching

Searching numerical data lets you find information, such as boiling points and molecular weights. When performing a numerical search, you can specify the range using a hyphen between the values at either end of the range. Ranges may also be indicated using inequality operators (<, >) together with the AND operator.
In numerical searching, you provide a decimal value or range in the query. If you provide a single value, the number of significant digits determines the precision of the search. A hit is any value that rounds off to the value specified in your query.
Here are a few examples of numerical queries:

 

Entry in a Numerical Field

Results Found

78.15

Values from 78.15 and greater, inclusive.

78

Values from 78 and greater inclusive.

89.5-90.5

Values from 89.5 to 90.5.

90.05-91.05

Values from 90.05 to 91.05.

90-100

Values from 90 through 100, inclusive.

>90 and <100

Values from 90 through 100, exclusive.
