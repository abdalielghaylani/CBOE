COEHitListService

 

The Search results of the search operation using COESearchService are persisted to a database as a HitList.  The HitList history is automatically maintained by the COE applications. HitLists can be declared either as private or public. If a HitList is declared as private it can only be accessed by the user who has created it. Alternatively, you can declare a HitList as public to make it accessible to all users.[c1]  The COEHitListService enables you to maintain the records of the COEHitListBO objects into the COEHitList table of the database.

 

The various properties of the COEHitListBO object are:

 

  • ID
  • Name
  • DateCreated
  • Description
  • UserID
  • IsPublic
  • FormGroup
  • HitListType
  • NumHits

 

Using this service, you can perform the following operations on COEHitListBO objects:

 

  • Create a new COEHitListBO object
  • Modify an existing COEHitListBO object
  • Delete an existing COEHitListBO object
  • Fetch the records of a COEHitListBO objects
  • Create a SAVE COEHitlistBO object from an existing TEMP COEHitListBO object
  • Perform subtraction operation on two HitLists.
  • Perform intersection operation on two HitLists.
  • Perform union operation on two HitLists.
