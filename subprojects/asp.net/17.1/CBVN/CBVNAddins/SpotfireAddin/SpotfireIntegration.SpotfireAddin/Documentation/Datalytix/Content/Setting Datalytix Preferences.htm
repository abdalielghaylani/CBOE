Setting Datalytix Preferences

In order to manage performance, there are three user-defined preferences:
Compress Data – controls whether data is sent in a compressed (True) or non – compressed (False) mode. By default, the data is compressed.
Maximum Row Limit - limits the number of rows returned by the query.
Page Row Count – controls the data page size. For example, if the Page Row Count is set to 100 and the data table contains 1000 records, Datalytix will fetch 100 records at a time, and repeat this process 10 times.
Use Remoting – controls whether the application will use remoting services (better performance) to connect to the database. If set to True, remoting services will be used (default configuration). If set to False, the application will use the web service to connect to the database.
 
To set the Datalytix Preferences:
1. Open the Spotfire client, and logon as a Spotfire Administrator. From the Tools menu, select the Administration Manager sub-menu item.
 

 
The Administration Manager window opens.
 

 
2. Click on the Preferences tab.
 

 
3. Select the Group Name to which the preferences should be applied.
 

 
4. Select the Preferences tab in the right hand panel. Expand the Datalytix category in the tree.
 

 
5. Select the Configuration sub-category in the tree, and click on the Edit button.
 

 
The Edit Preferences dialog opens. From here, the user can specify the Preference parameters.
 

 
6. Enter thePreference parameter(s).
7. Click OK to save and close the dialog.
8. Click Close to exit the Administration Manager.
Note: When searching, if a member of the Group for which the preference is set attempts to load more than the Maximum Row Limit, a message will appear indicating that the configured limit has been exceeded.
