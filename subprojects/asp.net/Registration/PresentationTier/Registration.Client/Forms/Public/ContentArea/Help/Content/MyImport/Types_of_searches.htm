Types of searches

You can search for records based on any fields in the Search Temporary Records page by specifying the required search criteria. In Registration Enterprise 11.0.1, the different types of searches you can perform include:
Structure search
Text search
Numeric search
Formula search
Date search
Boolean search

Structure search

Structure searching lets you search for records based on their chemical structures. Registration Enterprise 11.0.1 provides three types of structure searches:
Substructure searching
Full structure searching
Exact searching

Substructure searching

Substructure searching is the default structure search type. It retrieves structures that meet the following criteria:
Match the basic chemical structure specified
contain additional attachments at open bond positions
Tip: To narrow down or broaden search, attach different features such as atom lists and variable bond types to the structure using ChemDraw plug-in.

Full Structure Searching

Full structure searching retrieves structures that match the structure specified in the query, without considering additional attachments. Isotopes are recognized as hits.

 

Example of Full structure searching

Exact Searching

Exact searching retrieves structures that are identical to the structure specified in the query. Isotopes are not recognized as hits.

Setting search preferences

Setting the search preferences lets you control the manner in which the results are displayed. These settings are global for a user throughout the application.
To access the search preference options:
Click Options attached to the structure window. The Search Preferences dialog box appears.
General tab

 

The various options under the general tab include:
Hit any charge on heteroatom
allow uncharged atoms in the query to match charged atoms in the target.
Reaction query must hit reaction center
allow the reaction centers in the query to overlap with the reaction centers in the target.
Hit any Charge on Carbon
allow uncharged carbon atoms in the query to match charged carbon atoms in the target.
Permit extraneous fragments in full structure(exact) searches/Permit extraneous fragments in reaction full structure(exact) searches
perform exact structure search.
Query Fragments can overlap in target selected
fragments can be overlapped.
Tautomeric
allow to find tautomers in the query.
Similarity search
retrieves chemicals that have structural features similar to the one specified in the query. It does not match specific atoms or bonds. It is useful if you have a general idea of the types of compounds you are looking for, but do not know the precise target compund. You can specify the degree of similiarity. A lower value requires the target to be less similar.
Stereochemistry tab

 

The various options under the stereochemistry tab include:
Tetrahedral stereo center hits
Set the Tetrahedral stereo center hits.
Select Same to match the target exactly.
Select Either to match same or opposite configuration at the center of the target.
Select Any to match any target.
Select Thick bonds represent relative stereochemistry to match a relative relationship between centers. When activated, any thick bond drawn in the reference compound is interpreted as relative stereochemistry when searching or comparing possible duplicates
Double bonds hits
Set the Double bonds hits.
Select Same to match the target exactly.
Select Any to match any configuration in the target.

Text search

In text searching, use wildcard character(*) to broaden the search and use ‘=’to narrow to exact wording. The following table lists a few examples of text search.
 

Text entered

Records retrieved

*ben

Retrieves records that start with "ben", but may have any characters after. Retrieves benzene and benzofuran, but not bromobenzene.

*ben*

Retrieves records that have any character before "ben" and any character after "ben" in the name field. Retrieves benzene, bromobenzene, and benzofuran

=Benzene

Retrieves the exact text specified. Retrieves Benzene.

Ben*chl

Retrieves compounds with "ben" at the beginning, any character after "ben" and before "chl" and any character after "chl". Retrieves Benzyl chloride.

Numeric search

You can retrieve data such as boling points and molecular weights using numeric search. When performing a numeric search, you can specify a range using a hyphen between the values at either end of the range. Ranges may also be indicated using inequality operators(<,>) together with the AND operator.
The query is a decimal value or range. If a single value is given, the number of significant digits determines the precision of the search. A hit is any value that rounds off to the query.
The following table lists a few examples of numeric search.
 

Value in numeric field

Records retrieved

Values from 78 and greater inclusive.

78.15

Values from 78.15 and greater, inclusive

89.5-90.5

Values from 89.5 to 90.5

90.05-91.05

Values from 90.05 to 91.05

90-100

Values from 90 through 100, inclusive

>90 and <100

Values from 90 through 100, exclusive

Formula search

Formula searching lets you search molecular stochiometry, either exactly or in the range you specify. Formula queries consists of:
Element symbols
Element count or ranges
Element symbols may be one or two letters. Symbols may be in upper case or lower case. Symbols may be repeated. For example, CH3CH3 is interpreted as C2H6. To avoid ambiguities:
Capitalize the symbols properly and insert spaces between elements
Use the Periodic Table to enter formulas
Element counts are single integers or ranges. If a count is omitted, it is assumed to be 1. Parentheses may be used to group elements and apply a count to the entire group. For example, (CH2)3 is interpreted as C3H6. Spaces or non-alphanumeric characters other than parentheses are ignored.
Note: Formula searches are completely non-structural: CH3CH2OH matches both dimethyl ether and ethyl alcohol because both compounds have the same condensed formula: C2H6O.
The following table lists a few examples of formula search.

 

Formula entered

Records retrieved

C6H6

Compounds with 6 carbon and 6 hydrogen, plus any number of other elements. Retrieves C6H6, C6H6N2O.

=C6H6

Retrieves compounds with exactly 6 carbon and 6 hydrogen and no other elements

C6

Retrieves compounds with 6 carbon plus any number of other elements

C6 N1-3

Retrieve compounds with 6 carbon and one to three nitrogen, plus any number of other elements

Date Search

Date searching lets you dearch data based on the date field. You can specify a range using a hyphen between the values at either end of the range. Ranges may also be indicated using inequality operators(<,>) together with the AND operator.
The following table lists a few examples of date search.
 

Date entered

Records retrieved

Apr 11, 1971

Retrieves all entries with the exact date of April 11, 1971

Mar 31, 1971-July 1, 1971

Retrieves all entries from Apr 1-June 30 1971

>4/11/71

Retrieves any record after April 11, 1971

>4/11/71 AND <7/21/71

Retrieves any entry between April 11 and July 21, 1971

You can perform boolean search using boolean operators AND, OR, and NOT.
The following table lists a few examples of boolean search.
