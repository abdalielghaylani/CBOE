Managing hit lists

You can restore, save, edit, or delete any hit list generated from a query.

Saving hit lists

You can save the list of records from a query and retrieve it later for viewing. If the database has been updated since the query was run, the retrieved hitlist will be outdated and you will need to rerun the query to get an accurate list.
To save a hit list:
1. Perform a search. For more information, see "Searching databases".
2. Go to Hit lists> Save. The save hitlist window appears.
3. Enter a List Name and Description for the hitlist.
4. To enable others to view the hitlist, select the Make Public check box.
5. Click OK. The hitlist is saved.

Restoring hit lists

You can display the last created hit list or a saved hit list. To restore the last created hit list, go to Hit lists> Restore>Last. The hit list is displayed in List view.
To restore a saved hit list:
1. Do one of the following:
Go to Hit lists>Restore>Saved.The restore hitlist window appears.
Go to Hit lists>Manage. The manage hitlist window appears.
2. Select the hit list from the drop-down list in the window.
3. Select one of these options:
Replace current list

The saved list is displayed in list view as it was saved.

Intersect with current list

A hit list is displayed, which contains only the hits found in both the saved list and the current list.

Subtract from current list

A hit list is displayed, which contains all the hits in the current list, not including hits of the saved list.

Union with current list

A hit list is displayed, which contains all the hits in the saved list as well as those in the current list.

4. Click Restore.The hit list is displayed in List view.

Editing a hit list

Editing a hit list lets you enter new identifier information for the hit list, such as a new name and description.
To edit a hit list:
1. Go to Hit lists>Manage. The manage hitlist window appears.
2. Select the hit list from the drop-down list in the window.
3. Click Edit. The edit hitlist window appears.
4. (Optional) Modify the List Name and Description fields.
5. (Optional)To enable/disable others’ ability to view the hitlist, select/deselect Public List as needed.
6. Click OK.

Deleting a hit list

Only saved hit lists can be deleted. Hit lists in the history remain until the history is cleared. To learn more about clearing the history, contact your system administrator.
To delete a hit list:
1. Go to Hit lists>Manage. The manage hitlist window appears.
2. Select the hit list from the drop-down list in the window.
3. Click Delete. A message box appears, prompting you to confirm the deletion.
4. Click OK to delete the hit list.

Hit list history

You can retrieve a hitlist that was executed in the current session, even if the hitlist was not saved. The history is cleared periodically, based on the application configuration. Consult your system administrator for more information about history clearance.
To restore a hitlist in your history:
1. Go to History> Hit lists. The History window appears.
2. Select the hit list from the drop-down list.
3. Click Restore.

Marking records

You can mark records (added to a progressive hit list) in either the Details or List views. Marked hits are user specific and can be retrieved at any time.
To mark a record:
In List View: Click the Mark Record button corresponding to the record that you want to mark.
In Details View: Click the Mark Record button in the upper left hand corner of the record that you want to mark.
Note: The record is marked and the button changes to Unmark Record button. To remove the record from the marked records list, click the Unmark Record button.
To view the marked records, go to Marked Hits > Show Marked. The marked records are loaded into the current hit list and displayed.
To delete marked records, go to Marked Hits > Clear Marked. The marked records are deleted.
