Fragment Information

When a component is a chemical compound, only base structure is drawn and the rest of the structure information about the compound is specified as Fragment Information, which differs from one batch to another.

Example:

Salts and solvates are the most typical examples of fragment types. A component, as in the case of hydrated salts, may have both salt and solvate as fragment types.

Using Data Loader, you can import up to two fragments per component into Registration Enterprise. The fragments can be of any fragment type including customer dependent types, not just salts and solvates.

The Fragment Information section consists of these fields:

  • Fragment 1: Select the first fragment type.
  • Fragment 1 Equivalents: Specify the fragment equivalent.

NOTE: An equivalent of a substance is defined as the amount of it that combines with one mole of hydrogen atoms.

  • Fragment 2: Select the second fragment type
  • Fragment 2 Equivalents: Specify the fragment equivalent.
