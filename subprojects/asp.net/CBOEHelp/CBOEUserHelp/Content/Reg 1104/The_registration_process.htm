The Registration Process

The registration process includes the following steps in order:

  1. The Submitter (typically a chemist/biologist), enters chemical data into Registration. The data is stored in a temporary database. The submitter can search, edit, or delete the data as desired.
  2. The Reviewer checks and approves records in the temporary table for registration. This is applied when "ApprovalsEnabled" is set to True in the Registration system settings.
  3. The Registrar (typically a senior chemist or lab manager with SET_APPROVED_FLAG and TOGGLE_APPROVED_FLAG privileges) reviews and registers submissions to the registry. The data is then stored in the permanent database.
  4. The Registrar (typically a senior chemist or lab manager with SET_LOCKED_FLAG and TOGGLE_LOCKED_FLAG privileges) locks registered records making them un-editable and indelible. This is applied when "LockingEnabled" is set to True in the Registration system settings.

Note that both features, Approvals and Locking, are not related. You can have a system with one of the following settings:

  • ApprovalsEnabled=True and LockingEnabled=False
  • ApprovalsEnabled=False and LockingEnabled=True
  • ApprovalsEnabled=True and LockingEnabled=True
  • ApprovalsEnabled=False and LockingEnabled=False

In registration system:
  • Registration assigns a unique number to each record in the permanent table to identify it. The registration ID can be used to search the record.
  • Registration checks for duplicate records whenever a component is registered in the permanent table. Multiple cases are available here. for more details please refer to the "Managing Duplicates" topic.
