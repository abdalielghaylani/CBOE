Loading Compounds - InvLoader

Inventory Loader is an application that lets you import compounds from XLS files, CSV files, and ChemBioFinder database and load them into the Inventory Enterprise or Registration Enterprise applications.

Inventory Loader runs only on the computer that has access to the ChemBioOffice Enterprise application server.

Most often, the compounds to be loaded into the Inventory Enterprise or Registration Enterprise applications are stored in a SDFile. In such situation, you can load the compounds by converting the SDFile into a ChemBioFinder database. You can convert the SDFile into a ChemBioFinder database using the ChemBioFinder application.

The Inventory Loader application can import compounds in three ways, which are:

  • Into a series of new plates
  • Into containers
  • Directly into Inventory Enterprise

When importing compounds, you need to select the appropriate fields from the source database. However, Inventory Loader eliminates the need to select the fields repeatedly by allowing you to save the information as mapping templates and reuse them later.

Loading Compounds into Plates

When you use Inventory Loader to load compounds into plates, new plates are created in Inventory Enterprise and they are then loaded with the compounds imported from the selected database file.

To load compounds into plates using Inventory Loader:

  1. Start the Inventory Loader application. The Import Database screen of the Inventory Loader wizard, appears as:

  2. Click Next. The Login to the Inventory Web Server screen appears as:

  3. Enter the name of the ChemBioOffice server, your user name and password in the Server, User ID, and Password text boxes, respectively. In order to enable the secure connection check the "Use SSL (https:)" checkbox.

    4. Note: Ensure that the user name that you have entered in the preceding window has sufficient permissions to create plates and load compounds in it. Also, ensure that the user name has sufficient permissions to access Registration Enterprise, if you want to register the compounds into the Registration Enterprise database.

  4. Click Next. The Select Database screen appears as:

    5. Note: If an error message appears, stating that no plate locations or formats are configured in Inventory Enterprise, then you need to create a location that can hold plates and/or create a valid plate format, in Inventory Enterprise.

  5. Ensure that the Load Structures into Plates radio button is selected. Browse and locate the database file from which the compounds are to be imported. The file can be a CSV file, XLS file, or a ChemBioFinder database.

    6. Note: Choosing a format other than ChemBioFinder database will not have any impact on the behavior of the application; field mappings and registration options will work in exactly the same manner for all three formats.

    However, in case of importing data from a XLS file with multiple worksheets, you will have to select the actual worksheet name that contains the data you wish to import.

  6. Select the Register the new compounds in the CambridgeSoft Chemical Registration System checkbox if you want to register the compounds into the Registration Enterprise database. When you select this checkbox, Inventory Loader verifies whether or not you have sufficient permissions to register compounds into Registration Enterprise. If Inventory Loader finds your authentication information valid, a grid containing the Registration Enterprise fields displays in the Registration Info screen. Otherwise, an error message is displayed

  7. Click Next, the Select Fields screen appears as:

Note: If an error message appears, stating that the database cannot be opened then you need to upgrade the database to the current version of ChemBioFinder. To upgrade the database, open the database using the ChemBioFinder and follow the prompts.

Note: If the ChemBioFinder database contains more than one table, you are provided with a drop-down list of the table names. You need to select the table with which you want to map the fields from the drop-down list. Typically, the name of the table with which the fields are mapped is MolTable.

In the Select Fields screen:

  • The Data Type column facilitates data validation by providing you prior information about the type of data to be entered.
  • The Value column displays the plate fields.
  • The Field Name column lets you set the value of the fields. You can set a field either by mapping it with a column of a table of the ChemBioFinder database or by specifying a value for it in the Default column.

While setting the plate fields, consider the following aspects:

  • It is not necessary to set Row and Column fields as well as the Well Coordinate field. However, if you set the Row field, it is necessary to set the Column field.
  • If some columns of the table with which you are mapping the fields contain plate identifiers, then map those columns with the Barcode and Supplier Barcode fields. Map the column that contains numeric plate identifiers with the Supplier Plate Number field.
  • It is necessary to set one of the Supplier Barcode, Supplier Plate Number, or Barcode fields and one of the Well Coordinate or Row and Column fields, otherwise, the compounds may not load into the plates correctly.

Note that InvLoader supports the Row Level Security(RLS). When your system has the RLS set at the Batch Level, you will be able to see Batch Project field instead of Project. Project option appears only when your system has one of the following settings: RLS=off OR RLS=Registry level. When RLS=Batch, this allows you to select batch project's related information from either an input file (or) from the user interface. Note that it is required to enable the Registration Integration by checking the "Register the new compounds in the CambridgeSoft Chemical Registration System" checkbox (Please refer to the step 6 above for more details).

There are four identifier types:

  • Reg Identifiers
  • Batch Identifiers
  • Structure Identifiers
  • Component Identifiers

The identifiers existing in the mapping dialog appear in their corresponding sections in Registration.

For each identifier you need to select the identifier type and the identifier value. Below is an example for a Reg Identifier mapping:

  • Double-click on the "Default" cell corresponding to the "Reg Identifiers" row. A popup containing all available Reg Identifiers appears. If there are no available identifiers then the popup appears empty.

  • Select the required identifier(s) using the ‘>’ and ‘<’ buttons and click Ok.
  • Double-click on the "Default" cell corresponding to the "Reg Identifiers Value" row. The following popup appears, displaying the available values:

  • To add values, type the value(s) in the text field and click on Add button.
  • After adding the value(s), click on OK, the value(s) get updated in the mapping dialog.
  • To edit a value, highlight it and click on Edit button. This allows you to make necessary changes.
  • To remove a value, highlight it and click on Remove button.
  • To replace a value, highlight it, enter a new value in the text field and then click on Replace button.

Note: by default the four identifiers appear in the InvLoader irrespective of the Registration settings. The mapped identifiers appear in Registration only if it is enabled before the loading via: Customize Registration > System Settings > Advanced tab.

Fragments and Equivalents

A fragment can be a salt or solvate associated with the compound to be registered. Fragments are available for mapping only when you opt to register the compound and create plates or containers. You can select single/multiple fragment(s) either from the invLoader user interface or map it from the input file. When multiple fragments are mapped, one of the following separators should exist between the fragments: |(pipe), ;(semicolon) or ,(comma) . If the user maps multiple fragments from the input file, then the separator used in the input file should be the same one selected in the Registration Integration dialog. The default separator character is the | pipe.

For each fragment selected or mapped you should its corresponding equivalent value. This can be either selected from the invLoader user interface or mapped from the input file.

To map fragments from the Select Fields dialog:

  • Double-click on the Default cell corresponding to the Fragment row
  • A dialog displaying the available fragments appears. Select the fragment(s) you want and click on OK button.

To select equivalents from the Select Fields dialog:

  • Double-click on the Default cell corresponding to the Equivalents row
  • A dialog displays allowing you to type the equivalent values. Type the value you want in the text field and click on Add button. Repeat this action as many times as needed in order to specify equivalents to all the fragments you have selected.

To map fragments from an input file:

Note: Some properties should be in a specific format in the sdf or cfx file:

Properties Format
Caution , Storage , Value1 ; Value2 ; Value3 ;
Identifiers , Fragments , Salts , Solvents , Impurities , Non Standard Salts

These properties should be listed in one property in the sdf file with the following format :

IdentifiersRegistry:Identifier1:Value1; Identifier2:Value2; Separator

IdentifiersCompound: Identifier1:Value1; Identifier2:Value2; Separator

IdentifiersBatch: Identifier1:Value1; Identifier2:Value2; Separator

Salts:Salt:Stochiometrie; Separator

Solvents: Solvent:Stochiometrie; Separator

Impurities: Impurity:MW:Stochiometrie; Separator

Fragments: Fragment:Equivalent; Separator

Non Standard Salts: Salt:MW:Stochiometrie; Separator

Example: (Separator = END)

IdentifiersRegistry:Alias:edyR;Old external codes:edy2;END
IdentifiersCompound:Alias:edyC;Old external codes:edy2;END
IdentifiersBatch:Alias:edyB;Old external codes:edy2;END
Salts: Nitrate:3;Pyridine salt:4;END
Solvents: Hydroxide:4;END
Impurities: Impurity1:4:2;END
Fragments: Borohydrate:3;END
Non Standard Salts: Salt:2:3;edys:4:5;END

Note:

The separator is set in the add-in configuration in Reg addins. No order is needed and also user can add only properties needed.

Example:

IdentifiersRegistry:Alias:edyR;Old external codes:edy2;END
Salts: Nitrate:3;Pyridine salt:4;END

In the above example only Registry identifiers and salts on batch are added to the record.

Loading Compounds into Containers

To load compounds into containers:

  1. Start the Inventory Loader application. The Import Database screen of the Inventory Loader wizard, appears.
  2. Click Next. The Login to the Inventory Web Server screen appears.
  3. Enter the name of the ChemBioOffice server, your user name and password in the Server, User ID, and Password text boxes, respectively.
  4. Click Next. The Select Database screen appears.
  1. Click the radio button, Load Structures into Containers.
  2. Click the Browse button locate the database file from which the compounds are to be imported. The file can be a CSV file, XLS file, or a ChemBioFinder database.
  1. Click Next. The Registration Info screen appears. Select the Register the new compounds in the CambridgeSoft Chemical Registration System check box if you want to register the compounds into the Registration Enterprise database. When you select this check box, Inventory Loader verifies whether or not you have sufficient permissions to register compounds into Registration Enterprise. If Inventory Loader finds your authentication information valid, a grid containing the Registration Enterprise fields is displayed in the Registration Info screen. Otherwise, an error message is displayed.

  2. The Update Selections dialog displays. Note that this dialog displays only while loading structures into containers. Select one of the following options:
    • No Updates: will skip the available containers
    • Update to non-null values only: will update only properties filled in the next step
    • Update including non-null to null: will update all properties and set as null empty properties if they were filled for the existing container

If you select the "Update to non-null values only" or "Update including non-null to null" options, you will be able to update the registered compounds properties along with inventory compounds, as shown below:

Select one of the following options:

  • No compound updates: will skip updating the registration data for any available compounds
  • Update compounds to non-null values only: will update only the properties filled in the mapping fields section (E.g. Synonym, Comments, Purity, Fragment etc..)
  1. Click Next. The Select Fields screen appears.
  2. Set the fields in the Select Fields screen, as required. In case you have saved the mapping information earlier as a mapping template, you can select the fields by clicking the Load button and selecting the saved mapping template.
    • Value column: is the list of properties (Plate to Plate / Container to Container / Registration to Registration). (Same label of the field present in inventory or registration)
    • Data Type:is the value type of the property
    • Field Name:is the column to map properties values from the .cfx file to the loaded compounds in inventory or registration (e.g. in the screen shot below the properties Location Barcode and Container Name values are mapped from the .cfx file properties selected in the Field Name column to the compounds loaded into Inventory and same if we choose registration properties)
    • Default column:is used to set a default value in a certain property to be loaded to all compounds

The screen shot below shows an example of a one to many property which is called "List" in InvLoader.

  1. After mapping and setting all needed values, click Next. The Log Plate XML screen appears as:

  2. Log Plate XML is the last stage in the InvLoader wizard and here the user can check the log data checkbox to get a log file with the mapped data columns or errors if the loading fails. Click on Advanced Options button to display the Options dialog allowing you to set the time out of the server while loading and to define the number of records that is sent to the server in each round-trip. The three checkboxes below are used to save the log files for the containers/compounds/plates in xml format during the load process for troubleshooting purpose.

  3. Click Next. The Ready to Import screen appears.
  4. Click Finish. The import process starts.
  5. Click Finish again when the import process is complete.

Loading Compounds Directly

To import the compounds directly into the Inventory Enterprise and Registration Enterprise databases:

  1. Start the Inventory Loader application. The Import Database screen of the Inventory Loader wizard, appears.
  2. Click Next. The Login to the Inventory Web Server screen appears.
  3. Enter the name of the ChemBioOffice server and your user name and password in the Server, User ID, and Password text boxes, respectively.
  4. Click Next. The Select Database screen appears.
  5. Click the Load Structures Only radio button.
  6. Click the Browse button to locate a database and click Next.
  7. Select the Register the new compounds in the CambridgeSoft Chemical Registration System check box if you want to register the compounds into the Registration Enterprise database. When you select this check box, Inventory Loader verifies whether or not you have sufficient permissions to register compounds into Registration Enterprise.
  8. The Select Fields screen appears. Set the fields displayed in the Value column. You can set a field either by mapping it with a field of ChemBioFinder database table in the Field Name column or by specifying a default value for the field in the Default column.
  9. Click Next. The Log Plate XML screen appears.
  10. Select the Log data to the following location check box if you want to log the output for future reference.
  11. Specify the location at which you want to log the data in the text box displayed below the Log data to the following location check box.
  12. Click Next. The Ready to Import screen appears.
  13. Click Finish when the import process is completed.
