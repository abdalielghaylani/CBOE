Send Records to Application Server

After validation, the prepared Registry Records are transferred from the client to the application server. A fraction of the record load time is consumed by this data transfer step. The highest gain in this step can be achieved by avoiding the cost of moving data between a remote client system and the application server. This can be accomplished by executing the bulk load client from the application server. If the bulk load must be performed from a remote client, there can be modest performance gains by adjusting the ChunkSize parameter in the COEDataLoader application configuration file. This settings determines how name registry records are packaged by the client they are sent to the server. A value of up to 200 may beneficial for client systems that have a high latency connection to the application server. Increasing to a very high value can cause a very large data packet to be sent and could result in errors if the HTTP message size limits established by the application server are exceeded.

Recommendations:

  • Perform bulk load from the application server
  • Set transfer ChunkSize parameter between 10-200

Details:

For DataLoader, edit COEDataLoader.exe.config and modify the JobChuckSize parameter.

For command line DataLoader edit COEDataLoader.exe.config and modify the ChunkSize parameter.
