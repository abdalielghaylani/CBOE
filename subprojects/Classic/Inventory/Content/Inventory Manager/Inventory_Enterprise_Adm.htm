LPR Printing Service

The Inventory Enterprise 11.0 schema has been modified to maintain a list of network printers and LPR print definitions that you can customize to suit your needs. The LPR service accepts a text file and sends it to the actual print device.

In case of Zebra printers, this text file includes Zebra printing language codes that specify various parameters of the text, such as font, position, and size.

To add flexibility in how the text file is generated, a new table named “inv_lpr_definition” is created to maintain a list of print definition files. Each definition entry has a header, label, and footer file.

The printing API will read each of these three files, set the fields that are specific to any location/container/plate, and then concatenate the three files together into a single print file that is sent to the printer.
