Assigning the Label to Printer

Once an LPR definition has been created, you need to group that definition with a label type and assign it to a particular printer. You also need to provide additional information that the LPR service needs in order to communicate with the printer itself. This information is managed through the new inv_label_printers table, which is named “Label Printers” within the Manage Tables GUI. The GUI exposes the following fields, all of which are required:

• Server name, which is the DNS name or actual IP address to the printer (or print server, as the case may be).
• Queue name, which may or may not exist on the Zebra printer (or print server).
• LPR definition foreign key, which points to the LPR definition ID set up previously.
• Report type foreign key, which points to one of the valid inv_reporttypes records (1 = container barcode, 7 = plate barcode, 9 = location barcode).
• Display name, which is a 200-character string that will show to the user when printing.
• Is Available, which is a 1/0 boolean flag to tell the application whether a particular printer is to be displayed to the user or not This is useful when a printer is offline for service.

When the user chooses to print a label within the GUI forms, the application takes the following actions:

1. Determines which kind of barcode is to be printed (containers, plates, or locations).
2. Selects all of the available printers that match that label type.
• If no LPR printers are available, the standard Access snapshot report generator is displayed to the user.
• If LPR printers are available, the user is given a choice as to which kind of label they wish to generate, as shown in this figure:

 

 

In this context, the user is given an option between “Network” printers or “My local” printers. The former utilize the new LPR print mechanism, while the latter utilize the existing legacy Access snapshot report generator, which requires printing through the user’s local desktop.

• If the user chooses one of the local printer reports, the default Access snapshot viewer is displayed as normal.
• If the user chooses one of the network printer reports, the following actions occur:
• Based on the LPR definition, the application reads in the appropriate header, label, and footer files from the file system.
• All recognized special tags in all three file templates are replaced with the appropriate values from the object data result set and stored in memory.
• A single print job is created by concatenating the header, label, and footer data together into a single output stream.
• The single output stream is written out to a unique temporary file, and this file is sent to the LPR service.
• The LPR service reads the temporary file and handles communication with the printer, as specified by the Label Printer record in the database.
• User retrieves barcode.
