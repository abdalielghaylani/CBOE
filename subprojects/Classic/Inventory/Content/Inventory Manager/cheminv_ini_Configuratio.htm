cheminv.ini Configuration File

The ChemInv.ini file is found at the following location: <webroot>/ChemOffice/cheminv/config

This file defines many parameters that allow Inventory Enterprise to function properly, including the definitions of formgroups (which define all forms) in the application. Most of this file should not be edited since editing this file in the wrong place could cause major problems when running the Reg System application.

NOTE:   It is the responsibility of a site's administrator to propagate any changes made to ChemBioOffice Enterprise application files to future versions of the application. New versions of the ChemBioOffice Enterprise use default configurations.

GLOBALS

ABOUT_WINDOW

Example: ABOUT_WINDOW = Inventory Enterprise 9.0.0.37

Description: The text that appears in the about window describing the name of the data view.

Options: Up to one line of text.

ADO_CONNECTION_NAMES

Example: ADO_CONNECTION_NAMES = base_connection

Description: Names of all ado connection definition sections in the ini file. This list must be comma delimited with no spaces. If the connection name is not here, it will not be loaded by the application. An ADO definition section describes variables for connecting to data source via ADO. In general there is only the default definition "base_connection_group", since most dataviews access the same database. However, this is flexible and allows more definitions then just the default. ADO_Connection_Names are referenced by individual tables.

Default: base_connection

Options: Comma delimited list of connection definitions found in the ini file.

CFW_INSTANCE

Example: CFW_INSTANCE = cheminv

Description: If equal to first, specifies that this dataview should share an instance of ChemBioFinder with first dataview specified as the GlobalBaseDB key in the cfserver.ini file. If this key is empty (or set to new) then the first database in the DB_NAMES list is used.

Default: cheminv (first name in db_names is used)

Options: first | new

CHEM_CONNECTION_NAMES

Example: CHEM_CONNECTION_NAMES = cheminv_cfw_form

Description: Names of all ChemBioFinder connection definition sections in the ini file. This list must be comma delimited with no spaces. If the connection name is not here, it will not be loaded by the application. A ChemBioFinder connection definition section describes variables for connecting to ChemBioFinder for searching of chemical information. The default definition "base_cfw_form" is normally present (unless there is no chemical information). In dataviews that deal with reaction databases, there may be three (or more) definitions, one for the reaction, one for the solvent and one for the catalyst. (e.g.config/chemprep.ini file in the chemrxn application).

Default: cheminv_cfw_form

Options: Comma delimited list of ChemBioFinder connection definitions found in the ini file.

DB_RECORD_COUNT

Example: DB_RECORD_COUNT = 2000

Description: The number of records in the database. This appears in all input and result windows in the upper left corner next to TotalRecords.

Default: 2000

Options: Any integer (equal to the number of records in the database).

DB_TYPE

Example: DB_TYPE = STRUC

Description: States the type of database.

Default: STRUC

Options:

DISPLAY_NAME

Example: DISPLAY_NAME = ChemInventory

Description: The name used in global search screens to identify the dataview.

Default: ChemInventory

Options: Any text

FIELD_MAP_GROUPS

Example: FIELD_MAP_GROUPS = gs_field_map_group,global_substanceselect_field_map_group

Description: Names of field map group definition sections in the ini file. This must be comma delimited, with no spaces. If the field group names are not here, they will not be loaded by the application. Field Map groups specify field mapping used by global search. If global searching is not used, the FIELD_MAP_GROUPS (default group created by the wizard) can be set to NULL. FIELD_MAP_GROUPS are reference by formgroups.

Default: gs_field_map_group,global_substanceselect_field_map_group

Options: Comma delimited list of field map group definitions found in the ini file.

FORM_GROUPS

Example: FORM_GROUPS = base_form_group,containers_form_group,containers_np_form_group, substances_form_group,add_record_form_group,gs_form_group,global_substanceselect_form_group

Description: Names of form group definition sections in the ini file. This must be comma delimited, with no spaces. If the form group names are not here, they will not be loaded by the application. Form_groups specify a sub set of information including the input and result forms to use, the fields that can be searched, the table group to use for searching as well as additional information that will be described in later sections. Form_groups are referenced by hyperlinks from the main.asp page and carried throughout all functions and subroutines within ChemBioOffice Enterprise.

Default: base_form_group,containers_form_group,containers_np_form_group, substances_form_group,add_record_form_group,gs_form_group,global_substanceselect_form_group

Options: Comma delimited list of form group definitions found in the ini file.

MAXHITS

Example: MAXHITS = 1000

Description: The number of hits returned before a Get More button will appear.

Default: 1000

Options: Any integer

SUBFORM_VIEW_NAMES

Example: SUBFORM_VIEW_NAMES = NULL

Description: Internal Setting.

Default: NULL

Options: Should not be edited.

TABLE_ALIASES

Example: TABLE_ALIASES = inv_compounds,inv_containers,inv_locations,inv_synonyms,synonym, inv_plates,inv_wells,well_compounds_alias,inv_wells_alias,reg_numbers,reg_numbers_alias

Description: Names of all table definition sections in the ini file. This list must be comma delimited with no spaces. If the table name is not here, it will not be loaded by the application. Table definition sections describe all the information needed by ChemBioOffice Enterprise for searching tables.

Default: inv_compounds,inv_containers,inv_locations,inv_synonyms,synonym, inv_plates,inv_wells,well_compounds_alias,inv_wells_alias,reg_numbers,reg_numbers_alias

Options: Comma delimited list of tables

TABLE_GROUPS

Example: TABLE_GROUPS = base_table_group,containers_table_group,substances_table_group

Description: Name of table group definition sections in the ini file. This must be comma delimited, with no spaces. If the table group names are not here, they will not be loaded by the application. A table group specifies information about the order of searching tables, what the base table (searched via ADO) is and what the molecule table (searched via ChemBioFinder) is. The majority of applications use the default table group named "base_table_group". Table_groups are referenced by Form_Groups.

Default: base_table_group,containers_table_group,substances_table_group

Options: Comma delimited list of table group definitions found in the ini file.

ADO Connection Definitions

CONN_TYPE

Example: CONN_TYPE = "FILE NAME"

Description: Specifies the type of connection string.

Default: FILE NAME

Options: DBQ | DSN | OLEDB

CONNECTION_PASSWORD

Example: CONNECTION_PASSWORD = "login_required"

Description: Contains the password for the connection. The way the password is entered is different for different data sources. Otherwise, this entry are probably left empty.

Default: "login_required"

Options: N/A

CONNECTION_STRING

Example: CONNECTION_STRING = C:\Inetpub\wwwroot\ChemOffice\ChemInv\Config\ChemInv.udl

Description: The connection string for the CONN_TYPE specified.

Default: C:\Inetpub\wwwroot\ChemOffice\ChemInv\Config\ChemInv.udl

Options: A Connection String

CONNECTION_TIMEOUT

Example: CONNECTION_TIMEOUT = 30

Description: The amount of time in minutes before a connection will automatically close upon no activity. We suggest 15 to 30 minutes.

Default: 30

Options: An integer indicating a number of minutes.

CONNECTION_USERNAME

Example: CONNECTION_USERNAME = "login_required"

Description: Contains the user name for the connection.

Default: "login_required"

Options: N/A

COMMAND_TIMEOUT

Example: COMMAND_TIMEOUT = 30

Description: The amount of time in minutes before a command object will automatically close upon no activity. We suggest 15 to 30 minutes.

Default: 30

Options: An integer indicating a number of minutes.

DBMS

Example: DBMS = ORACLE

Description: The database server type.

Default: ORACLE

Options: Should not be edited.

PWD_KEYWORD

Example: PWD_KEYWORD = password

Description: This a password descriptor used to build the connection string.

Default: password

Options: Should not be edited

USERID_KEYWORD

Example: USERID_KEYWORD = uid

Description: This a user ID descriptor used to build the connection string.

Default: uid

Options: Should not be edited
