Creating Registry Records

You can create a registry record that has either a single component or multiple components. Pure substances consist of a single component whereas substances like mixtures consist of multiple components. You can add multiple component to a single registry record.

To create a registry record with single component:

1. Click Submit new record in the Registration home page. The Submit Record page appears.

 

2. Fill in Registry Information by selecting values in the fields Projects, Identifiers, and Prefix. If the registry record belongs to more than one project, click Add Project and select the required value. If the registry record is assigned to more than one identifier, click Add Identifier and select the required identifier type.
NOTE:Projects, Identifiers, and Prefix are defined by the administrator in the table editor. For more information, refer to the Administrator guide.
3. Fill in the Registry Custom Properties by entering comments associated with the record in the Registry Comments field.
4. Draw a structure in the window. There are three options available:
• Drawn Structure(Above) is selected by default.
• Select No Structure if the component does not have a structure associated with it, as in the case of proteins.
• Select Unknown Structure if you don’t know the structure and you want to keep the structure field blank for the moment. You can later add the structure to the record.
• Non Chemical Content is used to paste a graphic image in the structure box. To do so, Right-click inside the structure box and select Edit>Paste from the context menu. The image appears in the structure box.
5. Fill in Component Information by selecting values in the fields Component Identifiers, Component comments, Stereochemistry comments, and Prefix. To assign more than one identifier, click Add Identifier and select the required identifier type.
6. Fill in Fragment Information by selecting the Fragment Type, and Code. The fragment type could be a be salt, solvate or any other fragment type defined by the administrator. A component, as in the case of hydrated salts, may have both salt and solvate as fragment types. To select more than one fragment type, click New Fragment. Select the code based on the fragment type selected and enter the equivalents number for each fragment. All other fields under Fragment Information is filled automatically.
NOTE:For more information on defining new fragment types and adding new fragments, refer to the Registration Enterprise Administrator guide.
7. Fill in the Batch Information by selecting values in the fields Chemist, synthesis Date, Units, and other required fields.
8. Click Submit. The record is added to the temporary table and the temporary record ID appears. Click Done.
NOTE:To create a registry record with multiple components, follow steps 1-6 of creating records with a single component above and then follow the instructions below.

You can create registry records with multiple components by either adding a new component or selecting a component already registered in Registration.

To create a registry record with multiple components:

1. Follow the steps 1-6 of creating records with a single component mentioned above.
2. Click Select Component to add a registered component. The Search Component form appears. Enter the Registration # and click Search. If the registration # is valid, the component information is loaded. To add a new component, click Add Component. A new component icon appears in the New Registry tree structure on the left-hand side of the window.

 

3. Fill in the required fields for the component and click Add component to add more components. To delete any component you have added, click Remove Component.
4. Click Continue when you are done adding components.
5. Specify the Batch Component Custom Properties.
6. Click Submit to add the record to the temporary table. The record is added to the temporary table and the temporary record ID appears. To view the component details, click the corresponding component in the New Registry tree structure.
