Creating the Chemical Inventory Database

Note: The sql scripts provided have been designed to be executed with the command line version of SQL*Plus. This scripts contain substitution variables which will not be proccessed correctly when run from SQL*Plus Worksheet. The easiest way to execute the scripts with SQL*Plus is to double click the provided command files.

Creating a Blank ChemInv Database

  1. Browse to <webroot>/cheminv/config/oracle_intall_scripts/create_blank_cheminv_db/
  2. Create the Database:
    • If a previous version of ChemInv is installed
      1. Execute the Drop_ChemInvDB.cmd file.
      2. Execute the Create_Blank_ChemInvDB.cmd file.
    • If no previous version of ChemInv is installed
      1. Execute the Create_Blank_ChemInvDB.cmd file.
  3. Execute the Import_ChemInvDB_PickLists.cmd file.
  4. Use the ODBC Control Panel to create a system DSN
    • Open the ODBC Control Panel 
      • NT4.0 Server: Located in control panel
      • Windows 2000 Server: Start Menu >Programs>Administrative Tools>Data Sources
    • Select  System DSN tab
    • Click Add
    • Double Click on Oracle ODBC Driver
    • Fill in the following information:
      • Data Source Name: Chem_Inv
      • Description: Chem_Inv
      • Service Name: <Oracle Service Name> for your target database  
      • User ID: cheminvdb2

 Creating a Test ChemInv Database

A test ChemInv database is provided with the application to help users get acquainted with the system without having to populate their own inventory data.  The procedure for importing the test ChemInv data described below requires a preexisting ChemInv database schema.  If no ChemInv database exists on the sever follow the instructions above to create a blank database before proceding.

Note: Loading the test database will overwrite any existing inventory data.  Make sure to backup your inventory database before proceding. 

  1. Browse to /cheminv/config/oracle_intall_scripts/create_test_cheminv_db/
    • execute the Create_Test_ChemInvDB.cmd file.
    • execute the Import_Test_ChemInvDB.cmd file.
  2. Copy the mst and msi files from \chemInv\config\oracle_install_scripts\create_test_chemInv_db to the location specified by the STRUC_DB_PATH entry in the cheminv.ini file
  3. Reset IIS.  This should be done after importing a new database.

Testing the ChemInv Database

  1. Browse to /cheminv/config/. Open ChemInv.udl.  Change the Data Source to match your Oracle service name. Set the password to "oracle". Click the Test Connection button
  2. Use the Oracle ODBC Test tool to verify connectivity to the database via the Chem_Inv DSN
