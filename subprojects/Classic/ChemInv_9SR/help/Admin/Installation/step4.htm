Setting up the Report Writer

Note: If you have already configured Reports.exe to run with any ChemOffice Webserver application, you do not have to repeat these steps, or the steps to setup PDF support.  Just make sure reports.exe is running.

There are two options in setting up the report writer:

  • Run it as a service
  • Run it locally on start up

It is only necessary to complete one set of instructions.

Running Reports.exe as a Service

Reports.exe is the application used by the ChemOffice WebServer products to write reports. The application must be running on the server in order for a report to be made, so in order to guarantee this, it is best to set up reports.exe to run as a service. Doing this will not require any additional steps to start running reports.exe other then starting the server.

Four files are necessary to set up the Report Writer as a service. These files are included in a self-extrating zip file:
<webroot>/Chemoffice/webserver_source/cfserverasp/RPT/reports_zip.exe

After locating the file:

  1. Double-click on reports_zip.exe.
    This creates a new directory (C:\Reports_As_Service) containing the files necessary to set up the service.
  2. Follow the instructions found in:
    C:\Rpt_As_Service\run_as_service_instructions.txt
    to finish setting up reports.exeto run as a service.
  3. Create a COM+ application for the ReportQ.dll.
    • Go to Start > Programs > Adminstrative Tools > Component Services.
    • Expand Component Services>Computers>My Computer node.
    • Expand COM+ Applications node.
    • Click on COM+ Applications folder and right click.
    • Select New > Application from the menu.
    • Click Next
    • Click the Create an Empty Application button.
    • Enter the name:  RPT in the text field.
    • Select Server application option
    • Click the Next button.
    • Select this user radio button.
    • Under User: enter “Administrator”
    • Under Password: enter the password for the local Administrator account
    • Confirm the password.
    • Click the Finish button.
    • The RPT application should appear in the COM+ Applications folder
    • Expand the RPT Package and left click on the Components folder.
    • Right click on the selected Components folder and choose New > Component.
    • Click Next
    • Click the Install new components button.
    • Browse to Program Files\ChemOffice2002\Common\dlls\REPORTQ.dll
    • Click open

    • The ReportQ component should appear in the Components folder.

  4. Make sure reports.exe is set to maximized on startup
    1. Double-click on reports.exe
      If the reports box opens, the application is set correctly.
    2. If the reports box does not open, only the icon appears in the task bar, double-click on the icon.
      This opens the reports box.
    3. Click Stop.
    4. Click Settings.
    5. Select Full Screen from the Display on Startup listbox.
    6. Click OK.

Running Reports.exe Locally

  1. Browse to <webroot>/chemoffice/webserver_source/cfserverasp/RPT
  2. Add a shortcut of the reports.exe to the Startup menu (in Documents and Settings) under ‘all users’
    • Note: The reports.exe program will automatically be excuted as soon as a user physically logs into the server console. The report writer will not work unless a user is constantly logged into the server.  To avoid the requirement of logging into the server please contact support@cambridgesoft.com to obtain instructions on how to install the reports.exe as a service.
