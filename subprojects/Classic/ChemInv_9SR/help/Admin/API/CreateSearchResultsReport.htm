CreateSearchResultsReport

Causes the RPT report writer to produce a printable search results report.  Search results reports contain a list of containers matching a preceding ChemInv appliation search.  A hitlist must exist on the server for the report to be run.  Hitllists are only persisted for the duration of the user session performing the search.  The function returns the file name of the report as saved to the reports directory.  Typically the URL to the reports directory is /cfserverasp/RPT/Reports/.  Check the invconfig.ini file for the location of the reports directory on your server.  

Syntax

/cheminv/api/CreateSearchResultsReport.asp?ReportName=adVarChar&HitListID=adNumeric&[ReportFormat=adVarChar]

Inputs

ReportName

The name of the report template as defined in the MS Access reports database.

HitListID

HitList_ID value indicating to the appropriate Hit List (previously generated by a search).

[ReportFormat]

The format of the printable report.  Allowed values include SNP (Access SnapShot format), PDF (Adobe Portable Document Format), RTF (Rich Text Format) and XLS (Microsoft Excel document).  Defaults to SNP.

Outputs

None

Example

A user creates a Search Results Report using the report called SearchResults2 and the hitlist with ID 7.  The report format will default to SNP.

/cheminv/api/CreateSearchResultsReport.asp?ReportName=SearchResults2&HitListID=7
