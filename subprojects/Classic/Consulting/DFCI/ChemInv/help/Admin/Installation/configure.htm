Admin Guide

Making Required Configurations

  1. Define the Data Source in the cs_security UDL by performing the following steps:
    1.  Browse to <webroot>/ChemOffice/ cs_security/config/.
    2.  Double-click cs_security.udl to open it.
    3. Change the Data Source to the name of your Oracle service. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
    4. Enter oracle as the password and click Test Connection.
    5. Click OK.
    6. Click OK.
  2. Define the Inventory Enterprise CS Security Data Source in the Inventory Enterprise cs_security UDL by performing the following steps:
    1. Browse to <webroot>/ChemOffice/ cheminv/config/.
    2. Double-click cs_security.udl to open it.
    3. Change the Data Source to the name of your Oracle service. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
    4. Enter oracle as the password and click Test Connection.
    5. Click OK.
    6. Click OK. 
  3. Define the Inventory Enterprise Data Source in the Inventory Enterprise UDL by performing the following steps:
    1. Browse to <webroot>/ChemOffice/ cheminv/config/.
    2. Double-click cheminv.udl to open it.
    3. Change the Data Source to the name of your Oracle service. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
    4. Enter oracle as the password and click Test Connection.
    5. Click OK.
    6. Click OK.
  4. Use ODBC Control Panel to create a system DSN for the Report Writer by performing the following steps:
    1. Select StartàProgramsàAdministrative ToolsàData Sources to open the ODBC Control Panel.
    2. Select System DSN tab.
    3. Click Add.
    4. Highlight Oracle ODBC Driver.
    5. Click Finish.
    6. Provide the following information:
  •  Data Source Name: Chem_Inv
  • Description: Chem_Inv
  • Service Name: <Oracle Service Name> for your target database. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
  • User ID: cheminvdb2
  1. Update the configuration files by performing the following steps:
    1. Browse to and open <webroot>/ChemOffice/cheminv/config/chemInv.ini. In the chemInv.ini file, ensure the following:
  • STRUC_DB_PATH entry agrees with the database path chosen in the installer. By default, the database path is C:\ChemOffice_data\Cheminv.
  • CONNECTION_STRING entry agrees with the application path chosen in the installer. By default, the database path is C:\Inetpub\wwwroot\ChemOffice\Cheminv\Config\Cheminv.udl.
  1. Browse to and open <webroot>/ChemOffice/cheminv/config/cfserver.ini. In the cfserver.ini file, perform the following tasks:
  • Set the ORA_SERVICENAME entry to the Oracle service name of your target server. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
  • Set the DBA_USERNAME to a valid DBA username. This provides limited DBA permissions to the user that can be used for creating new users or roles.  
  • Set the DBA_PWD entry to the password associated with the username entered in DBA_USERNAME.

  1. Browse to and open <webroot>/ChemOffice/cheminv/config/invconfig.ini. In the invconfig.ini file, ensure the following:

  • INV_SERVER_NAME points to the server hosting the Inventory Enterprise database. This is the server that is referred by the fully qualified domain name of the server, and not the Oracle Service name.

  • RPT_PATH entry agrees with the ChemOffice Enterprise installation path.

  • REPORT_DB_PATH entry agrees with the database path chosen in the installer. By default, the database path is C:\ChemOffice_data\ChemInv\Cheminv_reports.mdb.

  1. Setup barcode support by performing the following steps:

    1. Browse to <webroot>\ChemOffice\ ChemInv\Installation\Tbarcode.

    2. Run the Tbarcode.ocx installer. 

  2. Configure the Report Writer.

Note: You do not need to perform this step if the Report Writer is already configured while setting up some other application.

  1. Login as the user under whom the report service is running. This is the same user, whose user name was entered during the ChemOffice Enterprise installation.
  2. Set a default printer or install Win2PDF. Win2PDF is a PDF printer driver whose installer is found in chemoffice/cheminv/installation/win2pdf/w2pdf151.exe. You need to have license for installing Win2PDF.
  3. Launch MS Access to clear any initial dialogs for the user.

  1. Disable the integration of Inventory Enterprise if you do not want to integrate it with other ChemOffice Enterprise applications, by performing the following steps:
    1. Browse to and open <webroot>/ChemOffice/cheminv/config/invconfig.ini
    2. Set the REG_SERVER_NAME to NULL
    3. Set the ACX_SERVER_NAME to NULL
    4. Set the DOCMANAGER_SERVER_NAME to NULL

Note: If you want to integrate Inventory Enterprise with one or more than one ChemOffice Enterprise applications, leave these steps and move on to Step 5, Integrating With Other Applications.

  1. Perform an IISRESET.
