Integrating With Other Applications

Inventory Enterprise can be integrated with other ChemOffice Enterprise applications, such as Registration Enterprise, ChemACX, and DocManager Enterprise.

Integration With Registration Enterprise

To integrate Inventory Enterprise with Registration Enterprise:

  1. Browse to <webroot>/ChemOffice/cheminv/config/.
  2. Double-click ChemReg.udl.
  3. Change the Data Source to the name of your Oracle Service. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
  4. Click OK.
  5. Browse to and open <webroot>/ChemOffice/cheminv/config/invconfig.ini.
  6. Change the REG_SERVER_NAME entry to point to the server hosting Registration Enterprise. This is the server that is referred by the fully qualified domain name of the server, and not the Oracle Service name.
  7. Browse to <webroot>/ChemOffice/chem_reg/config/.
  8. Open Cfserver.ini.
  9. Change POST_MARKED_SEND_TO_PAGE=NULL to:
    POST_MARKED_SEND_TO_PAGE= http://IIS_SERVER_NAME/cheminv/gui/ImportFromChemReg.asp, where IIS_SERVER_NAME is the name of the IIS server that is hosting Inventory Enterprise.

Integration with ChemACX

Using Access Database

To integrate Inventory Enterprise with ChemACX using Access database:

  1. Browse to <webroot>/ChemOffice/cheminv/config/.
  2. Double-click AccessIni.
  3. Copy the following files from AccessIni to config:
  • chemacx.udl
  • cfserver.ini
  • invacx.ini
  1. Update chemacx.udl by performing the following steps:
    1. Browse to <webroot>/ChemOffice/cheminv/config/.
    2. Double-click ChemACX.udl.
    3. Make sure the location of the ChemACX database, ChemACX.mdb, is correct for Microsoft Access installations of ChemACX.
    4. Change the Data Source to the name of your Oracle Service for Oracle installations of ChemACX.
    5. Click OK.

  1. Update invconfig.ini by performing the following steps:
    1. Browse to and open <webroot>/ChemOffice/cheminv/config/invconfig.ini.
    2. Change the ACX_SERVER_NAME entry to point to the server hosting ChemACX. This is the server that is referred by the fully qualified domain name of the server, and not the Oracle Service name.

Note: If ACX_SERVER_NAME is set to NULL, the ChemACX integration is disabled.

  1. Browse to and open <webroot>/ChemOffice/chemacx/config/acxconfig.ini.
  2. Set EXPORT_TO_CHEMINV="True".

Using Oracle Database

To integrate Inventory Enterprise with ChemACX using Oracle database:

  1. Browse to <webroot>/ChemOffice/cheminv/config/.
  2. Double-click OraIni.
  3. Copy the files from OraIni to config
  • chemacx.udl
  • cfserver.ini
  • invacx.ini
  1. Double click chemacx.udl
  2. Select the Oracle provider name.
  3. Enter the Data Source name.
  4. Set the User name to CHEMACXDB and Password to oracle

CONN_TYPE="FILE NAME"

CONNECTION_STRING="C:\inetpub\wwwroot\ChemOffice\ChemInv\config\ChemACX.udl"

CONNECTION_TIMEOUT=15

COMMAND_TIMEOUT=30

CONNECTION_USERNAME="login_required"

USERID_KEYWORD="User ID"

CONNECTION_PASSWORD="login_required"

PWD_KEYWORD="Password"

DBMS=ORACLE

Integration with DocManager Enterprise

The user, who has to integrate Inventory Enterprise with DocManager Enterprise, should be assigned the INV_MANAGE_LINKS privilege. To integrate Inventory Enterprise with DocManager Enterprise:

  1. Browse to <webroot>/ChemOffice/cheminv/config/.
  2. Open invconfig.ini.
  3. Change the DOCMANAGER_SERVER_NAME entry to point to the Server hosting DocManager Enterprise. This refers to the fully qualified domain name of the server, not the Oracle Service Name.

Note: If DOCMANAGER_SERVER_NAME is set to NULL, the DocManager Enterprise integration is disabled.  

  1. Set SHOW_DOCMANAGER_LINK to 1.
