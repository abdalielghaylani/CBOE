CheckOutContainer

Checks out a container from its current location for subsequent use at a different location. This API also modifies the current user and status of the container, if specified. 

Syntax

/cheminv/api/CheckOutContainer.asp?LocationID=adNumeric&ContainerID=adNumeric&[CurrentUserID=adVarChar&DefaultLocationID=adNumeric&OwnerID=adNumeric&ContainerStatusID=adNumeric]

Inputs

Parameter Name Data Type Description Options Required? Table Name Database Field Name
LocationID adNumeric Specifies the ID of the location to which the container is to be checked out.   Y INV_CONTAINERS LOCATION_ID_FK
ContainerID adNumeric Specifies the ID of the container that is to be checked out.   Y INV_CONTAINERS CONTAINER_ID
CurrentUserID adVarChar Specifies the ID of the current user of the container.   N INV_CONTAINERS CURRENT_USER_ID_FK
DefaultLocationID adNumeric Specifies the ID of the current location of the container.   N INV_CONTAINERS LOCATION_ID_FK
OwnerID adNumeric Specifies the ID of user to which the container is to be assigned, when it is checked out.   N INV_CONTAINERS OWNER_ID_FK
ContainerStatusID adNumeric Specifies the ID of the status to be associated with the container, when it is checked out.   N INV_CONTAINERS CONTAINER_STATUS_ID_FK

Output

The ID of the location where the container is checked out. This ID is stored in the LocationID parameter.

Example

The following code allows you to check out a container with ID 12345 to a different location and change its status to In Use. The IDs of the current location, destination location, and current user of the container are 110, 111, and mlp, respectively. 

/cheminv/api/CheckOutContainer.asp?LocationID=111&ContainerID=12345&[CurrentUserID="mlp"&DefaultLocationID=110&ContainerStatusID="In Use"]
