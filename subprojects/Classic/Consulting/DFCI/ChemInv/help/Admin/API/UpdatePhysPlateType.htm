UpdatePhysPlateType

Updates a physical plate type.

Syntax

/cheminv/api/UpdatePhysPlateType.asp?PhysPlateID=adNumeric&GridFormatID=adNumeric&PhysPlateName=adVarChar&RowCount=adNumeric&ColCount=adNumeric&WellCapacity=Int&CapacityUnitID=adNumeric&ZeroPaddingCount=adNumeric[&RowPrefix=adVarChar&ColPrefix=adVarChar&RowUseLetters=adNumeric&ColUseLetters=adNumeric&NameSeparator=adVarChar&NumberStartCorner=adNumeric&NumberDirection=adNumeric&IsPreBarcoded=adVarChar&SupplierID=adNumeric]

Inputs 

Parameter Name Data Type Description Options Required? Table Name Database Field Name
PhysPlateID adNumeric Specifies the ID of the physical plate type to be updated.   Y INV_PHYSICAL_PLATE PHYS_PLATE_ID
GridFormatID adNumeric Specifies the ID of the grid format associated with the physical plate type.   Y INV_PHYSICAL_PLATE GRID_FORMAT_ID_FK
PhysPlateName adVarChar Specifies the name of the physical plate type.   Y INV_PHYSICAL_PLATE PHYS_PLATE_NAME
RowCount adNumeric Specifies the number of rows that the plate created using the new plate type will contain.   Y INV_GRID_FORMAT ROW_COUNT
ColCount adNumeric Specifies the number of columns that the plate created using the new plate type will contain.   Y INV_GRID_FORMAT COL_COUNT
RowPrefix adVarChar Specifies prefix for the plate rows.   N INV_GRID_FORMAT ROW_PREFIX
ColPrefix adVarChar Specifies prefix for the plate columns.   N INV_GRID_FORMAT COL_PREFIX
RowUseLetters adNumeric Specifies whether the rows are to be identified with numbers or letters. The rows are identified with letters if pRowUseLetters is set to 1. 1|0 N INV_GRID_FORMAT ROW_USE_LETTERS
ColUseLetters adNumeric Specifies whether the columns are to be identified with numbers or letters. The columns are identified with letters if pColUseLetters is set to 1. 1|0 N INV_GRID_FORMAT COL_USE_LETTERS
NameSeparator adVarChar Specifies the character that separates the row and column IDs. For example, '-' is the name separator for the following column and row IDs: A-1 and B-1.   N INV_GRID_FORMAT NAME_SEPARATOR
NumberStartCorner adNumeric Specifies the value with which the plate numbering is to be started.   N INV_GRID_FORMAT NUMBER_START_CORNER
NumberDirection adNumeric Specifies whether to fill the rows first or to fill the columns first.   N INV_GRID_FORMAT NUMBER_DIRECTION
IsPreBarcoded adVarChar Specifies whether the barcodes should be already defined for the plates created using the new physical plate type. The barcodes are already defined for the plates if pIsPreBarcoded is set to Y. Y|N N INV_PHYSICAL_PLATE IS_PRE_BARCODED
SupplierID adNumeric Specifies ID of the suppliers of the plates created using the new physical plate type.   N INV_PHYSICAL_PLATE SUPPLIER_ID_FK
CapacityUnitID adNumeric Specifies the ID associated with the unit in which the well contents are to be measured.   Y INV_PHYSICAL_PLATE CAPACITY_UNIT_ID_FK
WellCapacity adNumeric Specifies the capacity of the wells of the plates created using the new physical plate type.   Y INV_PHYSICAL_PLATE WELL_CAPACITY
ZeroPaddingCount adNumeric Specifies the number of digits with which the column and row numbers are to be padded. Usually, this parameter is set to 2.   Y INV_GRID_FORMAT ZERO_PADDING_COUNT

Output

The ID of the updated physical plate type, which is stored in the pPhysPlateID parameter. The data type of the pPhysPlateID parameter is adNumeric.

Example

The following code updates a physical plate type with ID 1020:

/cheminv/api/UpdatePhysPlateType.asp?PhysPlateID=1020&GridFormatID=1021&PhysPlateName=TestPlate&RowCount=8&ColCount=12&WellCapacity=12&CapacityUnitID=4&ZeroPaddingCount=2[&RowPrefix=B]
