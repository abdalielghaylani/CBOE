Conflict Resolution

When you attempt to create a new substance in Inventory Enterprise, a check is performed to ensure that the substance is not already there in Inventory Enterprise. This is done by ensuring that the unique fields of the new substance do not conflict with the unique fields of an existing substance. The unique fields for a substance are configured by the system administrator, however, by default, unique fields for a substance are name, chemical structure, CAS Registry number, and ACX number.

If the newly created substance already exists in the Inventory Enterprise, a conflict occurs and the following Conflict Resolution screen is displayed:

In the Conflict Resolution screen, the conflicting fields are marked in red. For example, in the preceding Conflict Resolution screen, the name of the substance is marked in red, as it is conflicting with the name of an existing substance.

The Conflict Resolution screen provides you with the following options to resolve the conflict:

  • Select an existing substance.
  • Edit the conflicting substance.
  • Create a duplicate substance and resolve the conflict later.
