Integration with Other Applications

ChemACX can be integrated with Inventory Enterprise and/or Registration Enterprise. 

Inventory Enterprise

  1. Update chemacx.udl.
    1. Browse to <webroot>/ChemOffice/cheminv/config/
    2. Double click ChemACX.udl
      • For Microsoft Access Installations of ChemACX: Make sure the location of the ChemACX database (ChemACX.mdb) is correct.
      • For Oracle Installations of ChemACX: Change the Data Source to the name of your Oracle Service
    3. Click OK.
  2. Update invconfig.ini.
    1. Browse to and open <webroot>/ChemOffice/cheminv/config/invconfig.ini
    2. Change the ACX_SERVER_NAME to point to the Server hosting ChemACX
      This refers to the fully qualified domain name of the server, not the Oracle Service Name.
      If set to NULL ChemACX integration will be disabled.
    3. Browse to and open <webroot>/ChemOffice/chemacx/config/acxconfig.ini
    4. Set EXPORT_TO_CHEMINV="True"
       
  3. If ChemACX is working with an Oracle database: update invacx.ini.
    1. Browse to and open <webroot>/ChemOffice/cheminv/config/invacx.ini
    2. Replace all occurrences of ACCESS with ORACLE.

Registration Enterprise

  1. Browse to <webroot>/ChemOffice/chemacx/config/
  2. Open acxconfig.ini
  3. Set REG_SERVER_NAME to the name of the server where the Registration Enterprise is installed.
  4. Set REGISTER_FROM_ACX to 1.
  5. Save the changes.
