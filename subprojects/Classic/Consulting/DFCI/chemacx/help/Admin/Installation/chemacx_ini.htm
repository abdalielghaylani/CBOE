Configuration File Settings

ChemACX.ini and acxconfig.ini files are located in /ChemOffice/ChemACX/Config

In ChemACX.ini

  1. Make sure STRUC_DB_PATH under [index_CFW_FORM] points to the location of the mst file 
  2. Make sure the connection string on the base connection points to the the ChemACX.udl file

acxConfig.ini:

The following is a list of parameters found in the acxConfig.ini file which can be used to adjust configuration settings. The parameter is shown in black, the useage is shown in blue, and an explanation about the parameter is shown in red.

[APPLICATION]

NP_SEARCH = “True|False” Default: “True”

Controls non-plugin search support. By default the application supports searching without a plugin. When set to True, a new link appears on the search form page pointing to a plain text search page. Non-plugin searches return chemical structures as GIF images rather than as cdx files.

[TABS]

FIELD_IN_SUPPLIER_TABLE = “Field Name” Default: “SupplierType”

Defines the field in the Supplier table that is used to segregate suppliers into the tabbed supplier lists.Must be a valid field name from the Supplier table. The field type is expected to be integer.

By default the ChemACX application looks for a field named SupplierType in the Supplier table of the ChemACX database. You can add an integer field to the Supplier table, give it any name, and use it to distinguish suppliers from one another. In this way, a single ChemACX database can support multiple ChemACX applications in which different suppliers appear in different tabs.

TAB1_VALUES = “An integer or coma delimited list of integers” Default: “1,2,3,4”

Determines which supplier(s) appear on the first tab. Value(s) are checked against the field of the Supplier table specified above.

TAB3_VALUES = “An integer or coma delimited list of integers” Default: “1”

Determines which supplier(s) appear on the third tab. Value(s) are checked against the field of the Supplier table specified above.

TAB1__GIF_ROOTNAME = “File Name Root” Default: “all_vendors”

TAB2__GIF_ROOTNAME = “File Name Root” Default: “your_favorites”

TAB3__GIF_ROOTNAME = “File Name Root” Default: “online_vendors”

These entries can be used to change the text that appears on the vendor selection tabs. Each tab uses two GIF image files; one with red background and white letters for the selected or “on” state, and another with white background and red letters for the not-selected or “off” state. The GIF files are named as rootname_on.gif and rootname_off.gif. To change the text that appears on a given tab two new GIF files must be produced and named following the above convention. Place the new .gif files in /chemoffice/chemacx/graphics directory. Then simply enter the new root name associated with each tab in the appropriate ini file entry.

TAB1_OPTION_TEXT= “String of max 20 Characters” Default: “ALL VENDORS:”

TAB2_OPTION_TEXT= “String of max 20 Characters” Default: “YOUR PREFFERED VENDORS:”

TAB3_OPTION_TEXT= “String of max 20 Characters” Default: “ONLINE VENDORS:”

Text appears at the top of the selection list when a given tab is selected. Keep it short. All upper case looks best.

[ROBOT]

USE_PRICE_ROBOT = “True|False” Default: “False”

This opens up the possibility of using a remote server to get latest pricing. This feature is not available in the current release. Must be set to false.

[DB]

ACX_DB_VERSION = “version number” Default: “”

FAVORITES_DB = “Database file path” Default: “”

This fields are for currently unimplemented features and should be left unchanged.

[PRICING]

PRICE_FIELD_IN_PACKAGE_TABLE= “Filed Name” Default: “PRICE”

DISCPRICE_FIELD_IN_PACKAGE_TABLE= “Filed Name” Default: “PRICE”

CSYMBOL_FIELD_IN_PACKAGE_TABLE= “Filed Name” Default: “CSYMBOL”

Determines which fields in the package table are used to look up prices. Use any valid field name from the ChemACX package table. The application can display up to two prices per product on the product grid. The DISC_PRICE field is only seen in the product grid. PRICE_FIELD is assumed to be the sale price and it is used in the shopping cart.

[SHOPPINGCART]

ORDER_AT_ACXCOM = “True|False” Default: “False”

If True, a “Create Order” button appears in the shopping cart which will forward the order to the ChemStore order processing system at chemacx.cambridgesoft.com where a purchase order is created. DO NOT SET TO TRUE UNLESS ADD_WITHOUT_PRICE = False.</FONT>

REQUEST_QUOTE = "True|False" Default: “False”

If true a “Request Quote” button appears which emails the shopping cart data to CambridgeSoft. CambridgeSoft will contact the user with a quote to procure the products via www.chemacx.com

EXPORT_TO_EXCEL = “True|False” Default: “False”

Allows contents of shopping cart to be exported to an excel template. The excel template will contain one worksheet per vendor in the shopping cart. Note Excel must be installed and properly configured on the server for this feature to work.

EXPORT_TO_WORD= “True|False” Default: “False”

Allows contents of shopping cart to be exported to an MS-Word template. Note MS-Word must be installed and properly configured on the server for this feature to work.

ADD_WITHOUT_PRICE= "True|False" Default: True

Allows all products to be added to shopping cart independent of price availability. Before setting to True, ensure that ORDER_AT_ACXCOM is set to false. The ChemStore order processing system cannot deal with orders that do not have a valid price included!

SHOW_LISTPRICE= "True|False" Default: False

Determines whether the list price appears in the product grid.

[SEARCHLOG]

LOG_SEARCHES= "True|False"

SEARCH_LOG_DB_CONNSTR= "Provider= SQLOLEDB; Data Source=; Initial Catalog= ; User Id= ; Password= "< /FONT>

This feature allows for searches to be logged to an ODBC database. It is not currently supported and should be left unchanged.

[ALTFILES]

NP_INPUTFORM= index.asp

ACXPROD_DISPLAY= acxprod_display.js

SUPPLIER_SELECTOR= supplier_select_cb.asp

Alternate code files to be used for customizing applications. Should be left unchanged.
