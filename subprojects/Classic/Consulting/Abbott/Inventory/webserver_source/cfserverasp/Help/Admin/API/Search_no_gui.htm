Search_no_gui

Using search_no_gui any application capable of reaching the ChemOffice WebServer via HTTP can request a search. The server will return a comma delimited hit list of values from a single, non-chemical, field in the base table specified by the return_fields parameter.

Note: All parameters can be passed either as Form (POST) or Querystring (GET) data.

Syntax

<ServerName>/<ApplicationName>/<ApplicationName>/<ApplicationName>_action.asp?dbName=adVarChar&dataaction=search_no_gui&struct_search_type=adVarChar[&return_fields=adVarChar&<Structure Field>= Base64cdxstring&<SearchField>=adVarChar&return_format=adVarChar] 

Inputs

dbName= <DatabaseName>

The name of the dataview ini file for the database being accessed.
Possible options : reg (for Registration System), cheminv (for Inventory Manager), chemacx, wdi, wda
Example:  dbname= chemacx

dataaction = search_no_gui

Must be set equal to "search_no_gui".

[Struc_search_type] =  Empty|Exact|Substructure

The type of structure search.  This parameter is equal to either exact or substructure .  The parameter defaults to exact.
Example: struc_search_type = exact

[return_fields] = <field name>

The fully qualified name of a non-chemical field from the base table.
By default the function returns the primary key of the base table.
Example: return_fields= Substance.ACXID

[<structure field>] =  <structure field value>

Where <structure field> is the fully qualified name of a structure field and <structure field value> is a Base64cdx string to search for.  
Note: The field name must match the structure field name on the ChemFinder form.
Note: Due to limitations on the maximum allowable length of the URL payload, lengthy data, such as base64cdx strings, must be posted as FORM payload.
Example: Substance.Structure= a lengthy base64cdx string.

[<search field>] = <search value>

Where <search field> is the fully qualified name of the search field to search on and <search value> is the value to search for.
Note: The field name must be declared in the ini file.
Example: Synonym.Name= benzene

[return_format] = Empty|server_rs

The default value is server_rs which is a comma separated list of values. 
ChemACX supports a second return format. Namely ACXXML. See http://www.chemacx.com/chemacx/chemacx/chemacx_postrequest.asp to see how xml formatted data can be retrieved from the server.
