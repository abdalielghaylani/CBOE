DataView INI settings (e.g. sample.ini)

Each ChemOffice WebServer Application can have more than one dataview. The config folder contains an ini file for the Application (cfserver.ini) and one ini file for each dataview. The dataviews that will load is determined by the list of DB_NAMES stored in the cfserver.ini. This allows enabling and disabling of data view loading. For information about the Application INI file see Application INI settings .

The Dataview files for each application are located in: <webroot>/ChemOffice/<app_name>/config
where <webroot> is your server's webroot and <app_name> in the application name.

This document discusses the sections and keys contained in a typical dataview ini file, the sample.ini file, for the Sample application. Settings for different applications may vary.  Please see an application's individual Admin guide for details.

It is important to note that it is the responsibility of a site's administrator to propagate any changes made to ChemOffice WebServer application files to future versions of the application.  New versions of the ChemOffice WebServer use default configurations.

When editing fileds in this file, it is important to be sure what is going to change because many of these settings are necessary for the application to function properly.

Descriptions are written in red while the text of the actual file is in black.  Fields shown in bold should not be edited.

[GLOBALS]
ABOUT_WINDOW=sample
the text that appears in the about window describing the name of the data view. Up to one line of text can be displayed.
DISPLAY_NAME=sample
the name used in global search screens to identify thedataview.
MAXHITS=100
the number of hits returned before a Get More button will appear.
CFW_INSTANCE=new
Values are "first" or "new"; first specifies that this dataview should share an instance of ChemFinder with first dataview specified as the GlobalBaseDB key in the cfserver.ini file. If this key is empty then the first database in the DB_NAMES list isused.
DB_RECORD_COUNT=288
number of records in database. This appears in all input and result windows in the upper left corner next to TotalRecords.
DB_TYPE=STRUC
TABLE_ALIASES=MolTable,Synonyms_r
Names of all table definition sections in the ini file. This list must be comma delimited with no spaces. If the table name is not here, it will not be loaded by the application. Table definition sections describe all the information needed by ChemOffice WebServer for searchingtables. See the MOLTABLE and SYNONYMS_R Definitions below.
SUBFORM_VIEW_NAMES=NULL
ADO_CONNECTION_NAMES=base_connection
Names of all ado connection definition sections in the ini file. This list must be comma delimited with no spaces. If the connection name is not here, it will not be loaded by the application. An ADO definition section describes variables for connecting to data source via ADO. In general there is only the default definition "base_connection_group ", since most dataviews access the same database. However, this is flexible and allows more definitions then just the default. ADO_Connection_Names are reference by individualtables.
CHEM_CONNECTION_NAMES=base_cfw_form
Names of all ChemFinder connection definition sections in the ini file. This list must be comma delimited with no spaces. If the connection name is not here, it will not be loaded by the application. A ChemFinder connection definition section describes variables for connecting to ChemFinder for searching of chemical information. In default definition "base_cfw_form " is normally present (unless there is no chemical information). In dataviews that deal with reaction databases, there may be three (or more) definitions, one for the reaction, one for the solvent and one for the catalyst. (e.g.config/chemprep.ini file in the chemrxn application).
TABLE_GROUPS=base_table_group
Name of table group definition sections in the ini file. This must be comma delimited, with no spaces. If the table group names are not here, they will not be loaded by the application. A table group specifies information about the order of searching tables, what the base table (searched via ADO) is and what the molecule table (searched via ChemFinder). The majority of applications use the default table group named "base_table_group ". Table_groups are referenced by Form_Groups
FORM_GROUPS=base_form_group,basenp_form_group,gs_form_group,add_record_form_group,drill_down_form_group
Names of form group definition sections in the ini file. This must be comma delimited, with no spaces. If the form group names are not here, they will not be loaded by the application. Form_groups specify a sub set of information including the input and result forms to use, the fields that can be searched, the table group to use for searching as well as additional information that will be described in later sections. The default formgroup is "base_form_group" this group is used most commonly. There may also be "basenp_form_group" which is used for input forms that do not have a plugin, "gs_form_group" used for performing global searches over many data views in the application; "add_record_form_group " which is reserved for future use but is not currently implemented and "drill_down_form_group" for displaying results for a drill down link contained in another form groups such as "base_form_group". Form_groups are reference by hyperlinks from the main.asp page and carried throughout all functions and subroutines within ChemOfficeWebServer.
FIELD_MAP_GROUPS=gs_field_map_group
Names of field map group definition sections in the ini file. This must be comma delimited, wiht no spaces. If the field group names are not here, they will not be loaded by the application. Field Map groups specify field mapping used by global search. If global searching is not used, the FIELD_MAP_GROUPS can be set to NULL. The default group created by the wizard. FIELD_MAP_GROUPS are reference by formgroups

####################
Ado connections specified in the ado_connections section of the globals section reference connection sections else where in the ini file. An ADO_Connection group is referenced by each table alias definition. The keys specify data necessary to create the connection to the database via ado.

[BASE_CONNECTION]
CONN_TYPE=DBQ
Specifies the type of connection string. Valid entries are "DBQ", "DSN" and "OLEDB".  See Connection Types for more information about your options.
CONNECTION_STRING=C:\Inetpub\wwwroot\ChemOffice\sample\database\sample.mdb;Driver= {MicrosoftAccessDriver(*.mdb)};DriverId=25;FIL=MS Access;
The connection string for the CONN_TYPE specified.
CONNECTION_TIMEOUT=30
The amount of time in minutes before a connection will automatically close upon no activity. We suggest 15 to 30 minutes.
COMMAND_TIMEOUT=30
The amount of time in minutes before a command object will automatically close upon no activity. We suggest 15 to 30 minutes.
CONNECTION_USERNAME=""
CONNECTION_PASSWORD=""
Contains the username and password.  If there is a password and you are using a DBQ or DSN then see the Connection Password Table .  The way the password is entered is different for different datasources.  Otherwise, these entries are probably left empty.

####################
ChemFinder connection names specified in the chem_connections section of the globals section reference connection sections elsewhere in the ini file. If a table contains chemical information stored by ChemFinder then CHEM_CONNECTION name is referenced by the table definition. The keys specify data necessary to create the ChemFinder connection.

[BASE_CFW_FORM]
STRUC_ENGINE=MOLSERVER
Specifies what chemical search engine is used. The only current option is MOLSERVER, soon to be expanded to include the Oracle Cartridge.
STRUC_FORM_NAME=sample
The unique portion of the ChemFinder form found in the app/dataview/cfwforms directory. There is both a short and long form. The long form is used for searches the short form is used for retrieving chemicals for display.
STRUC_DB_PATH=C:\Inetpub\wwwroot\ChemOffice\sample\database\sample.mst
The path to the mdb file that ChemFinder uses. Even if you are storing all data in Oracle or SQLServer, ChemFinder still uses an MDB file to link tables.
STRUC_TABLE_NAME=MolTable
The name of the table that the STRUC_FORM_NAME opens. This is often MOLTABLE, but may differ depending on how the original database structure was created.

####################
As with other items in the globals section the TABLE_GROUP key specifies individual table groups else where in the ini file. Table Groups specify information about what table is the base table for searching, what table holds the structures.

[BASE_TABLE_GROUP]
BASE_TABLE=MolTable
The name of the table whose ids are returned in a search. This is the base table for the dataview.
MOLECULE_TABLE=MolTable
The name of the table that is searched for chemical data. This may be the same or different then the basetable.
TABLE_SQL_ORDER=MolTable,Synonyms_r
The order in which the tables are added to a JOINclause.

[MOLTABLE]
TABLE_NAME=MolTable
RELATIONAL_FIELDS=ID;1,MOL_ID;1,Molname;0
PRIMARY_KEY=ID
SQL_SYNTAX=ACCESS
SELECT_KEYWORD=DISTINCT
SELECT_ADDITIONAL=NULL
SELECT_JOIN=MolTable.ID
SELECT_LINKS=MolTable.ID;1,MolTable.ID;1
INTER_TABLES=NULL
ADO_CONNECTION=base_connection
CHEM_CONNECTION=base_cfw_form
STRUC_FIELD_ID=MOL_ID

[SYNONYMS_R]
TABLE_NAME=Synonyms_r
RELATIONAL_FIELDS=ID;1,SYN_ID;1,Synonym_r;0
PRIMARY_KEY=ID
SQL_SYNTAX=ACCESS
SELECT_KEYWORD=NULL
SELECT_ADDITIONAL=NULL
SELECT_JOIN=Synonyms_r.SYN_ID=MolTable.MOL_ID
SELECT_LINKS=Synonyms_r.SYN_ID;1,MolTable.MOL_ID;1
INTER_TABLES=MolTable
ADO_CONNECTION=base_connection
CHEM_CONNECTION=base_cfw_form
STRUC_FIELD_ID=SYN_ID

####################
The form groups specified in the key, references individual form group sections elsewhere in the ini file. Form groups are a way for ChemOffice WebServer to specify different input and result forms and functionality via Form_group_flags. For instance the Base_form_group definition is the default definition for standard ChemOffice searching using the ChemDraw plugin, the basenp_form_group definition defines forms and functionality for non_plugin containing input forms. The gs_form_group is defined for global search functionality.

[BASE_FORM_GROUP]
INPUT_FORM_PATH=sample_input_form.asp
Name of search input form
INPUT_FORM_MODE=search
Mode that is set for the input form.
RESULT_FORM_PATH=sample_result_list.asp
Path to the result form. If you use standard ChemOffice WebServer terminology dataview_name & "_result_list.asp", the single record view is automatically determined as dataview_name & "_form.asp".
RESULT_FORM_MODE=list
The view that defaults when returning results. Options are list or edit. Edit is a historical name and means the single record view.
PLUGIN_VALUE=True
If there are structure, molecular weight or formula fields then this must be TRUE indicating that the ChemDraw applet should be written into the html output for the form. If specifying FALSE this means no applet code will be written in the file.
STRUCTURE_FIELDS=MolTable.Structure
Name of the table and field for the structure field in ChemFinder. For example, this is often MolTable.Structure. This is case sensitive relative to what is put into the input and result form. If no structure is present then enter NULL.
FORM_GROUP_FLAG=SINGLE_SEARCH
Options are "SINGLE_SEARCH" and "GLOBAL_SEARCH".
Use SINGLE_SEARCH for standard searching form groups; use "GLOBAL_SEARCH" when the form group is involved in global search functionality.
MW_FIELDS=MolTable.MolWeight
Name of the table and field for the molecular weight field in ChemFinder. This is often MolTable.MolWeight. This is case sensitive relative to what is put into the input and result form. If no structure is present then enter NULL
FORMULA_FIELDS=MolTable.Formula
Name of the table and field for the formula field in ChemFinder. This is often MolTable.FORMULA. This is case sensitive relative to what is put into the input and result form. If no structure is present then enter NULL
SEARCHABLE_ADO_FIELDS=MolTable.MOL_ID;1,MolTable.Molname;0,Synonyms_r.SYN_ID;1,Synonyms_r.Synonym_r;0
List of fields and their datatypes that are in the input form. This list is used by JavaScript to determine what fields are populated before the submit occurs. If there is no field listed here it will not be searchable. Syntax if tablename.fieldname;datatype with each full item separated by a comma. These items are case sensitive relative to what is in the input and result forms.  For a list of valid datatypes, please see the ADO_Fields Datatypes Table .
REQUIRED_FIELDS=NULL
SDFILE_FIELDS=TABLES:MOLTABLE,SYNONYMS_R
TABLE_GROUP=base_table_group
Name of the table group that this form_group references
NUM_LIST_VIEW=5
Number of records displayed in list view by default. Making this value higher then 20 makes output in list view that has structure, mw or formula fields very slow.

[BASENP_FORM_GROUP]
INPUT_FORM_PATH=sample_input_formnp.asp
INPUT_FORM_MODE=search
RESULT_FORM_PATH=sample_result_list.asp
RESULT_FORM_MODE=list
PLUGIN_VALUE=False
STRUCTURE_FIELDS=NULL
FORM_GROUP_FLAG=SINGLE_SEARCH
MW_FIELDS=MolTable.MolWeight
FORMULA_FIELDS=MolTable.Formula SEARCHABLE_ADO_FIELDS=Moltable.ROW_IDS;2,Synonyms_r.ROW_IDS;2,MolTable.MOL_ID;1,MolTable.Molname;0,Synonyms_r.SYN_ID;1,Synonyms_r.Synonym_r;0
REQUIRED_FIELDS=NULL
SDFILE_FIELDS=NULL
TABLE_GROUP=base_table_group
NUM_LIST_VIEW=5

[ADD_RECORD_FORM_GROUP]
INPUT_FORM_PATH=sample_add_form.asp
INPUT_FORM_MODE=add_record
RESULT_FORM_PATH=sample_add_form.asp
RESULT_FORM_MODE=add_record
PLUGIN_VALUE=True
STRUCTURE_FIELDS=MolTable.Structure
FORM_GROUP_FLAG=ADD_COMPOUND
MW_FIELDS=MolTable.MolWeight
FORMULA_FIELDS=MolTable.Formula
SEARCHABLE_ADO_FIELDS=MolTable.MOL_ID;1,MolTable.Molname;0,Synonyms_r.SYN_ID;1,Synonyms_r.Synonym_r;0
REQUIRED_FIELDS=MolTable.Structure:Structure,MolTable.Molname;0:Molecule Name
SDFILE_FIELDS=NULL
TABLE_GROUP=base_table_group
NUM_LIST_VIEW=5
