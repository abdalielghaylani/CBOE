Set up Global Security

Global security is used to limit users to particular parts of the applications. If you do not plan to use global security, you can skip this step.

Global security requires Oracle 8i or 9i and the appropriate client to be installed.

To set up global security:

  1. Browse to <webroot>/Chemoffice/ cs_security/config/Oracle_Install_Scripts/create_blank_cs_security_DB
  2. Execute the following file by double-clicking on it:
    Create_cs_security.cmd
    This will create the cs_security database.
    1. Press Enter.
    2. Enter your Oracle Service Name and press Enter. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
    3. Answer the rest of the questions, press enter to accept the defaults (shown in parentheses next to the questions).
    4. When the script is finished running, a log file opens in Notepad. Check the log file for connection errors. If there are no connection errors, close the log file (this closes the command prompt).
      CambridgeSoft recommends saving this log file for future reference in any accessible location.
  3. Browse to <webroot>/Chemoffice/ cs_security/config/
  4. Double-click cs_security.udl (to open it).
  5. Change the Data Source to the name of your Oracle service. For information about your Oracle Service Name, please see your Oracle DBA or consultant.
  6. Enter oracle as the password and click Test Connection .
  7. Click OK .
  8. Click OK .
  9. Browse to and open <webroot>/Chemoffice/config/chemoffice.ini
  10. Ensure USE_CS_SECURITY_APP=1 in [GLOBALS] section.
  11. This is the last step for installing ChemOffice WebServer.
  12. Restart your machine.

Test the installation of Global Security

  1. Click on the Global Login link on the ChemOffice Enterprise homepage.
  2. Login using the following username and password:
    • username: cssadmin
    • password: cssadmin
      The global security page opens.
