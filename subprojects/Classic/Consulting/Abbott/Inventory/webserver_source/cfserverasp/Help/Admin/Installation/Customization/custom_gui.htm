Changing The User Interface

It is important to note that it is the responsibility of a site's administrator to propagate any changes made to ChemOffice WebServer application files to future versions of the application.  New versions of the ChemOffice WebServer use default configurations.

Changing Buttons/Graphics

    To change the buttons/graphics for a single web application:

  1. Open the cfserver.ini file for the application and change the path specified for the graphics to a location within your web application.  For example /sample/graphics/navbuttons. NAV_BUTTONS_GIF_PATH=/cfserverasp/source/graphics/navbuttons/
  2. Save the cfserver.ini file.
  3. Copy the folder /cfserverasp/source/graphics/navbuttons too the directory entered in step 1.
  4. Change the files that you want, but keep the names the same. You now have your own custom set of graphics that will affect the web application whose cfserver.ini file you edited.

Removing Frames

If you remove frames you will lose some of the advanced functionality of the ChemOffice WebServer, but it will still function.  For instance, you will not be able to use mark hits and save query functionality and the browse buttons for a recordset will need to appear at the bottom of the main page so they will be updated properly.

  1. Remove frames in the default.asp file for the web application.
  2. Use the function calls in the navbar.asp file to retrieve the buttons you want to put in the main page. If you only want a single button rather then a group use the syntax GetButton('search') to display the search button, etc.
  3. Change the settings in the cfserver.ini file.  Be aware that by changing to no frames you will lose the user_info functionality which includes mark hits and save/restore query.
    MAIN_WINDOW=no_frames
    NAV_BAR_WINDOW=no_frames
    USER_INFO_WINDOW=no_frames

Changing the Frameset Definition

Within the default.asp file you can change the frame definition for a web application. The only thing to be cautious about is maintaining the names "navbar", "user_info" and main.  These are what the web server is looking for to perform many of it's basic and advanced functions in a frame environment.
