DataTypes (for the ADO_Fields entries)

 

Value DataType Example
0 text - Text field, formatted for SQL as a LIKE clause.  This is the default for text fields. User enters benzene,*benz, benz*
SQL is formatted as chemical_name LIKE "benzene"
1 integer User enters 1, 1-10, > 1.
SQL is formatted as:
REG_NUMBERS.Sequence_Number = 1 or REG_NUMBERS.Sequence_Number BETWEEN 10 AND 20, etc.
2 float Floats are always formmatted as if it were a range. Batches.SALT_MW BETWEEN 123.1500001 AND 123.2499999
3 text - Text field is formatted for SQL with "=" this has to be set manually as 0 is the default User enters benzene,*benz, benz*
SQL is formatted as: chemical_name= "benzene"
4 comma delimited numerical list - forces an IN clause   User enters 1,2,3,4 
SQL is formatted as reg_numbers.reg_number IN('AB-00001','AB-000002')
5 comma delimited textual list - forces an IN clause   User enters AB-00001,AB-00002
SQL is formatted as reg_numbers.reg_number IN(1,2,3,4)
6 boolean choices are 0 or 1
7 memo - forces text to be interperted as a LIKE

User enters benzene
SQL is formatted as chemical_name LIKE '%benzene%" 

8 date - type mm/dd/yyyy  User enters 10/20/2002 
SQL is formatted as:
ORACLE Reg_Numbers.Registry_Date=TO_DATE('10/20/2000','MM,DD,YYYY') ACCESS, SQLSERVER: Reg_Numbers.Registry_Date=#10/20/2000#
9 date - type dd/mm/yyy User enters 20/10/2002 
SQL is formatted as:
ORACLE Reg_Numbers.Registry_Date=TO_DATE('20/10/2000','DD,MM,YYYY') ACCESS, SQLSERVER: Reg_Numbers.Registry_Date=#20/10/2000#
13 Number(Real) - removes padding from a number. User enters " 1 ", it will be translated as "1". This is trimmed prior to formatting as type 1 .
20 Number(Real) - does NOT remove padding User enters " benzene", it will be translated as " benzene" (the space remains).
23 text - user input is exactly as is Unlike 3 , the user input is interperted as is.
25 FullText - ACCESS uses MS Index Server, Oracle uses Intermedia  
26 comma delimited textual list - forces an IN clause   Equivalent to 5 .
33 number range A special range search option used in the Merck Index.
35 converts boolean to NULL User enry of 1 and 0 is interperted as IS NOT NULL and IS NULL, respectiviely.
36 text - allows sql to be appended onto a sql string with an "AND " Mostly useful for subselects in a hidden field. the Hidden field named "alt_ids.identifier_type" might have the text as follows:
"alt_ids.identifier_type IN(select identifier_type form identifers where identifier_descriptor='chemical_name')" this would be added to the running sql clause as sql = sql & " AND " & alt_ids.identifier_type IN(select identifier_type form identifers where identifier_descriptor= 'chemical_name') Append_to_string = True
38 text - normalizes a name Used by chemindex. Formatted as type 0 after the Input is passed through multiple functions.  The search must be done against an appropriate field in the database which contains the normalized data.
39 text Same result as 38 but formatted like type 3 .
40 Full Text - full text search without using MS Index Server or Oracle Intermedia Formatted as type 0 after the Input is passed multiple functions. Performs a  full_text search without using MS Index Server or Oracle Intermedia:
strFieldDelimiter = "'" RangeSearch = True strDataType = 0
41 Full Text - full text search without using MS Index Server or Oracle Intermedia Formatted as type 7 after the Input is passed. through multiple functions. Performs a full_text search without using MS Index Server or Oracle Intermedia.
