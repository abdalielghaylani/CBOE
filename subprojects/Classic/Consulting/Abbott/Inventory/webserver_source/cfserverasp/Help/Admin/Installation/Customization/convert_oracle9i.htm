Converting an Access or SQLServer Based ChemOffice WebServer for use with Oracle 9i

Oracle 9i offers a tool called Oracle Migration Workbench that will help convert a legacy ChemOffice WebServer for use with Oracle.  Download the workbench application from:
http://otn.oracle.com/tech/migration/focusareas/access.html 
Be sure to download both the core and the appropriate plugin (Access or SQLServer). Note: The download directory for the following example will be assumed to be /ORA_WORKBENCH although it may differ depending on how you choose to install.

After unzipping the core download, browse to the install/win32 directory and click setup.exe. Do the same for the plugin after the core is installed.

The following instructions convert the Sample ChemOffice WebServer  application on a Windows2000 OS and WebServer2002 (or 2004). Earlier versions of WebServer do not fully support Oracle conversions.

Creating the SAMPLE_ORA application directory

  1. Browse to <webroot>\ChemOffice\sample, right-click and select copy.
  2. Paste the copy in the same directory. 
  3. Rename the copy sample_ora.
  4. Open <webroot>\ChemOffice\sample_ora\config\cfserver.ini and change APP_NAME=sample to APP_NAME = sample_ora.
  5. Right-click the directory
    <webroot>\ChemOffice\sample_ora\config\cfserver.ini
    and select properties.
  6. Click the Web Sharing tab.
  7. Select the Share This Folder radio button. Accept the defaults and click OK.
  8. Open sample_ora/config/sample.ini. Search and replace "ACCESS" or "SQLSERVER" with ORACLE.
  9. Open sample_ora\global.asa and change
    Application("appkey") = "sample"
    to
    Application("appkey") = "sample_ora"
  10. In the [BASE_CFW_FORM] section change:
    STRUC_DB_PATH=C:\Inetpub\wwwroot\ChemOffice\sample\database\sample.mst
    to:
    STRUC_DB_PATH=C:\Inetpub\wwwroot\ChemOffice\sample_ora\database\sample.mst
  11. In the [BASE_CONNECTION] section change:
    CONN_TYPE=DBQ
    CONNECTION_STRING=C:\Inetpub\wwwroot\ChemOffice\sample\database\sample.mdb;Driver= {MicrosoftAccessDriver(*.mdb)};DriverId=25;FIL=MS Access
    CONNECTION_USERNAME=""
    CONNECTION_PASSWORD=""
    to:
    CONN_TYPE=DSN
    CONNECTION_STRING=Sample_ORA
    CONNECTION_USERNAME=uid=sample
    CONNECTION_PASSWORD=pwd=oracle

Converting the database to Oracle 9i

  1. Browse to \ORA_WORKBENCH\0mwb\msaccess_exporter
  2. Double-click on mwb2000.mde
  3. Enter c:\inetpub\wwwroot\chemoffice\sample_ora\database\sample.mdb for the mdb file.c
  4. Click OK. The xml field is saved to the same directory.
  5. Next select Migration Workbench from Start/ORAHOME92/Configuration and Migration tools/Migration Workbench .
  6. Select Action/Capture Source Database .
  7. Click Microsoft Access from the list when presented.
  8. Click Next in the Capture Wizard Welcome Screen .
  9. Click Add XML file and locate the xml file
    \inetpub\wwwroot\chemoffice\sample_ora\database\sample.xml)
  10. Click Next. Use the System/Manager account or an account that has DBA privileges when prompted.  For the remainder of the wizard screen(s) take all the defaults presented. Their are two wizards one that converts the database objects and one that creates the schema and imports the data.  Follow the defaults for both.
  11. For the Name of the schema and password enter "Sample" and "Oracle" without the quotes respectively.

Create a DSN for the sample_ora

  1. Select Start/ORAHOME92/Configuration and Migration tools/Microsoft ODBC Adminstrator .
  2. Click the System DSN Tab.
  3. Click ADD .
  4. Select Microsoft ODBC for Oracle .
  5. In Datasource Name enter "Sample_ORA" without the quotes.
  6. In User Name enter "Sample" without the quotes.
  7. In Server enter the name of the service for the oracle instance you installed to.
  8. Click OK .

Edit the chemoffice.asp page

  1. Open the inetpub/wwwroot/chemoffice.asp page
  2. Locate  the following:

    <TR>
            <TD>
               <b><Ahref="/sample/default.asp?formgroup=base_form_group&amp;dbname=sample">Sample</a></b>
            </TD>
    </TR>

  3. Select copy, then paste below the text below.

  4. Change the pasted text to the following:

    <TR>
            <TD>
               <b><Ahref="/sample_ora/default.asp?formgroup=base_form_group&amp;dbname=sample">Sample_ORA</a></b>
            </TD>
    </TR>

 

That should be it. You can click the link from the chemoffice.asp page and be up and running with an Oracle back end database. To have this schema available to BioSAR, grant  select on all tables to the BIOSAR_BROWSER_ADMIN ROLE
