Setting CS Security Password Restrictions

In addition to allowing control over password validation rules via the cfserver.ini settings, cs_security also supports advanced password management rules such as:

  • Schedeluded password expiration
  • User account locks on failed logins
  • Prevention of password reuse

Password management functionality is controlled via the use of Oracle profiles.

The Cs_Security database script creates a default user profile called CSUserProfile. The CSUserProfile is set up by default with the following largely unrestrictive limits: 

FAILED_LOGIN_ATTEMPTS UNLIMITED 
PASSWORD_LIFE_TIME UNLIMITED 
PASSWORD_REUSE_TIME UNLIMITED 
PASSWORD_REUSE_MAX UNLIMITED 
PASSWORD_LOCK_TIME 1/96 --- means 15 minutes 
PASSWORD_GRACE_TIME UNLIMITED

The cs_seucurity interface will display the appropriate error messages when accounts are locked or expired. Users who login during their expired password grace period will be forced to change their password before they proceed. Users whose password is expired beyond the grace period will have to contact an administrator to have their account reinstated.

New user accounts created via the cs_security "Manage Users" interface will be automatically associated with the above CSUserProfile.
Note: CS schema owners and test user accounts will NOT be assigned the above profile because it is not desirable to force their passwords to expire.

Profile settings can be changed in Oracle by using: 

Connect system/manger@serviceName
ALTER PROFILE csuserprofile LIMIT 
FAILED_LOGIN_ATTEMPTS 3 
PASSWORD_LIFE_TIME 90 
PASSWORD_REUSE_TIME UNLIMITED 
PASSWORD_REUSE_MAX 3 
PASSWORD_LOCK_TIME 1 --- means one day 
PASSWORD_GRACE_TIME 30
;

The above example would lock user accounts after three failed login attemps; force users to change their password every 90 days; and force three password changes before a previously used password can be reused. Locked accounts would remain locked for one day. Users who fail to login for 30 days beyond the time their password expires would require an administrator to reinstate their account. For additional information on these settings refer to the Oracle SQL documentation.
