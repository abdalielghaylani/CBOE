Configure the Report Writer

Reports.exe is the application used by the ChemOffice WebServer products to write reports (i.e. for Inventory Manager and BioSAR Browser). If you do not plan to install and use this feature, you can skip this step.

The application must be running on the server in order for a report to be made, so in order to guarantee this, it is best to set up reports.exe to run as a service. Doing this will not require any additional steps to start running reports.exe other then starting the server. For this reason, by default the installer sets up the Report Writer service for you. 

Take the following additional steps to finish configuration:

  1. Login as the user that the report service is running under (this is the user name entered during the ChemOffice Enterprise installation).
  2. Set a default printer or Install Win2PDF (a PDF printer driver whose installer is found in the iInstallation folder of both Invnetory Manager and BioSAR Browser.  A license is required.).
    Note: Installing Win2PDF will give your users the ability to create reports in PDF format.
  3. Launch MS Access to clear any initial dialogs for the user.  If this step is not taken, the reportwriter will hang until the initial dialogs are cleared from Microsoft Access.  It is sufficient to clear the dialogs once.

Step 4: Test the Installation
