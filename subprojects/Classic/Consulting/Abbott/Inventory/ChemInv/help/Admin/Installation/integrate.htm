Integrate With Other Applications

Inventory Enterprise can be integrated with Registration Enterprise, ChemACX, and/or DocManager Enterprise.  The steps below will perform the integration:

Integration with Registration Enterprise

  1. Browse to <webroot>/ChemOffice/ cheminv/config/
  2. Double-Click ChemReg.udl
  3. Change the Data Source to the name of your Oracle Service.
    For information about your Oracle Service Name, please see your Oracle DBA or consultant.
  4. Click OK .
  5. Browse to and open <webroot>/ChemOffice/cheminv/config/invconfig.ini
  6. Change the REG_SERVER_NAME entry to point to the Server hosting Registration System.
    This refers to the fully qualified domain name of the server, not the Oracle Service Name.
  7. Browse to <webroot>/ChemOffice/chem_reg/config/
  8. Open Cfserver.ini
  9. Change POST_MARKED_SEND_TO_PAGE=NULL to:
    POST_MARKED_SEND_TO_PAGE= http://IIS_SERVER_NAME/cheminv/gui/ImportFromChemReg.asp
    (where IIS_SERVER_NAME is the name of the IIS server that is hosting Inventory Manager)

Integration with ChemACX

  1. Update chemacx.udl.
    1. Browse to <webroot>/ChemOffice/cheminv/config/
    2. Double click ChemACX.udl
    3. For Microsoft Access Installations of ChemACX: Make sure the location of the ChemACX database (ChemACX.mdb) is correct.
      For Oracle Installations of ChemACX: Change the Data Source to the name of your Oracle Service
    4. Click OK.
       
  2. Update invconfig.ini.
    1. Browse to and open <webroot>/ChemOffice/cheminv/config/invconfig.ini
    2. Change the ACX_SERVER_NAME to point to the Server hosting ChemACX
      This refers to the fully qualified domain name of the server, not the Oracle Service Name.
      If set to NULL ChemACX integration will be disabled.
    3. Browse to and open <webroot>/ChemOffice/chemacx/config/acxconfig.ini
    4. Set EXPORT_TO_CHEMINV="True"
       
  3. If ChemACX is working with an Oracle database: update invacx.ini.
    1. Browse to and open <webroot>/ChemOffice/cheminv/config/invacx.ini
    2. Replace all occurances of ACCESS with ORACLE.

Integration with DocManager Enterprise

  1. Browse to <webroot>/ChemOffice/cheminv/config/
  2. Open invconfig.ini
  3. Change the DOCMANAGER_SERVER_NAME to point to the Server hosting Document Manager.
    This refers to the fully qualified domain name of the server, not the Oracle Service Name.
    If set to NULL Document Manager integration with Inventory Enterprise will be disabled.
  4. Set SHOW_DOCMANAGER_LINK to 1.

Give any user who requires use of this integration the INV_MANAGE_LINKS privilege.
