CheckOutContainer

Checks Out a container to a new location and optionally assigns responsibility for the container to a new user and allows the user to change the status of the container. 

Syntax

/cheminv/api/CheckOutContainer.asp?LocationID=adNumeric&ContainerID=adNumeric&[CurrentUserID=adVarChar&DefaultLocationID=adNumeric&OwnerID=adNumeric&ContainerStatusID=adNumeric]

Inputs

LocationID

The Location_ID value of the location where the container is checked out to.

ContainerID

The Container_ID value of the container being checked out .

[CurrentUserID]

User_ID of the user how takes responsibility for the container.

[DefaultLocationID]

The current Location_ID associated with the location the container resides in normally.

[OwnerID]

User_ID of the owner of the container.

[ContainerStatusID]

ID associated with a particular container status for the container.

Outputs

Returns 1 if the process is sucessful.

Example

A user (ID mlp) checks out a container (ID 12345) currently in Location 110 and moves it to location 111, changing the status of the contaienr to In Use.

/cheminv/api/CheckOutContainer.asp?LocationID=111&ContainerID=12345&[CurrentUserID="mlp"&DefaultLocationID=110&ContainerStatusID="In Use"]
