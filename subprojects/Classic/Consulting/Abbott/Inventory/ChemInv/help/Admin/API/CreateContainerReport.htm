CreateContainerReport

Causes the RPT report writer to produce a printable container report.  Container reports are typically used to produce container labels which include a container barcode plus additional container attributes. The function returns the file name of the report as saved to the reports directory.  Typically the URL to the reports directory is /cfserverasp/RPT/Reports/.  Check the invconfig.ini file for the location of the reports directory on your server.  

Syntax

/cheminv/api/CreateContainerReport.asp?ContainerList=adVarChar&ReportName=adVarChar[&ReportFormat=adVarChar]

Inputs

ContainerList

A list of containers to be included in the container report. Use a comma to separate multiple Conatiner_ID values. Use a dash to express a range of Container_ID values.

ReportName

The name of the report template as defined in the MS Access reports database.

[ReportFormat]

The format of the printable report.  Allowed values include SNP (Access SnapShot format), PDF (Adobe Portable Document Format), RTF (Rich Text Format) and XLS (Microsoft Excel document).  Defaults to SNP.

Outputs

None

Example

A user wants to create a location report (LocationReport2) for containers in that location (IDs= 111, 112, and 113).  The default format is SNP.

/cheminv/api/CreateContainerReport.asp?ContainerList=111,112,113&ReportName=LocationReport2
