ReconcileContainer

Takes three lists of container ids: verified, awol and missing. Verified are updated as being where they ought to be. Awol are moved from where they used to be to the current location and marked as found during reconciliation. Missing are marked and moved as missing.

Syntax

/cheminv/api/ReconcileContainer.asp?LocationID=adNumeric&VerifiedContainerIDList=adVarChar[&AwolContainerIDList=adVarChar&MissingContainerIDList=adVarChar&CurrentUserID=adVarChar&OwnerID=adVarChar&ContainerStatusID= adNumeric]

Inputs

LocationID

Location ID for the location.

[VerifiedContainerIDList]

A comma delimited list of container IDs of containers verified as being in the specified location.
Required if AwolContainerIDList and Missing ContainerIDList are empty.     

[AwolContainerIDList]

A comma delimited list of container IDs of containers observer in the specified location, but not previously assigned to that location.
Required if VerifiedContainerIDList and Missing ContainerIDList are empty.     

[CurrentUserID]

The user ID for the curret user.     

[OwnerID]

The ID for the Owner of the containers.     

[MissingContainerIDList]

A comma delimited list of container IDs of containers assigned to the specified location but not verified as physically being in that location.
Required if AwolContainerIDList and VerifiedContainerIDList are empty.    

[ContainerStatusID]

The ID of the status that the containers will be given after the action is complete.         

Outputs

None

Example

A user has verified that containers 12345 and 12245 are in the specified location (ID = 101) but container 12225 is also in the location and wasn't previously assigned there. 

 
/cheminv/api/ReconcileContainer.asp?LocationID=101&VerifiedContainerIDList=12345,12245&AwolContainerIDList= 12225
