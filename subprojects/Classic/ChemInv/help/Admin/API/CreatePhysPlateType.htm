CreatePhysPlateType

Creates a physical plate type and returns the ID of the newly created physical plate type.

Syntax

/cheminv/api/CreatePhysPlateType.asp?PhysPlateName=adVarChar&RowCount=adNumeric&ColCount=adNumeric&WellCapacity=Int&CapacityUnitID=adNumeric&ZeroPaddingCount=adNumeric[&RowPrefix=adVarChar&ColPrefix=adVarChar&RowUseLetters=adNumeric&ColUseLetters=adNumeric&NameSeparator=adVarChar&NumberStartCorner=adNumeric&NumberDirection=adNumeric&IsPreBarcoded=adVarChar&SupplierID=adNumeric]

Inputs

Parameter Name Data Type Description Options Required? Table Name Database Field Name
PhysPlateName adVarChar Specifies the name of the physical plate type.   Y INV_PHYSICAL_PLATE PHYS_PLATE_NAME
RowCount adNumeric Specifies the number of rows in the physical plate.   Y INV_GRID_FORMAT ROW_COUNT
ColCount adNumeric Specifies the number of columns in the physical plate.   Y INV_GRID_FORMAT COL_COUNT
RowPrefix adVarChar Specifies the prefix to be used for physical plate rows.   N INV_GRID_FORMAT ROW_PREFIX
ColPrefix adVarChar Specifies the prefix to be used for physical plate columns.   N INV_GRID_FORMAT COL_PREFIX
RowUseLetters adNumeric Specifies whether to label rows with letters or with numbers. The rows are identified with letters if RowlUseLetters is set to 1. 1 | 0 N INV_GRID_FORMAT ROW_USE_LETTERS
ColUseLetters adNumeric Specifies whether to label columns with letters or with numbers. The columns are identified with letters if ColUseLetters is set to 1. 1 | 0  N INV_GRID_FORMAT COL_USE_LETTERS
NameSeparator adVarChar Specifies the character that separates the row and column IDs. For example, '-' is the name separator for the following column and row IDs: A-1 and B-1.   N INV_GRID_FORMAT NAME_SEPARATOR
NumberStartCorner adNumeric Specifies the value with which the plate numbering is to be started.   N INV_GRID_FORMAT NUMBER_START_CORNER
NumberDirection adNumeric Specifies whether to fill the rows first or to fill the columns first.   N INV_GRID_FORMAT NUMBER_DIRECTION
IsPreBarcoded adVarChar Specifies whether the barcodes should be already defined for the plates created using the new physical plate type. The barcodes are already defined for the plates if IsPreBarcoded is set to Y. Y | N N INV_PHYSICAL_PLATE IS_PRE_BARCODED
SupplierID adNumeric Specifies the ID of the suppliers of the plates created using the new physical plate type.   N INV_PHYSICAL_PLATE SUPPLIER_ID_FK
CapacityUnitID adNumeric Specifies the ID associated with the unit in which the well contents are to be measured.   Y INV_PHYSICAL_PLATE CAPACITY_UNIT_ID_FK
WellCapacity adNumeric Specifies the capacity of the wells of the plates created using the new physical plate type.   Y INV_PHYSICAL_PLATE WELL_CAPACITY
ZeroPaddingCount adNumeric Specifies the number of digits with which the column and row numbers are to be padded. Usually, this parameter is set to 2.   Y INV_GRID_FORMAT ZERO_PADDING_COUNT

Output

The ID of the newly created physical plate type. This ID is stored in the NewPhysicalPlateID parameter, the data type of which is adNumeric.

Example

The following code creates a new physical plate type with name NewPlateType, row count 8, column count 12, well capacity 1 ml, well capacity unit ID 9, and supplier ID 1123: 

/cheminv/api/CreatePhysPlateType.asp?PhysPlateName=NewPlateType&RowCount=8&ColCount=12&WellCapacity=1&CapacityUnitID=9[&SupplierID=1123]
