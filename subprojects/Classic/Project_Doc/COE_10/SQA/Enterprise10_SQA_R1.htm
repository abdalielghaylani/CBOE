1.0     Introduction

This document describes the Enterprise10 SQA Plan

2.0     Scope

2.1        The purpose of this document is to define the practices that will be used to assure software quality for ChemBioOffice Enterprise version 10, general release.

2.2 Development Constraints

This application should be written in such a way as to withstand a vendor audit. Appropriate documentation is required. Please see section 7, for a list of documentation deliverables for this release.

5.0     SQA Standards and Methodologies

5.1           Standards

The standards and methods are described in CambridgeSoft General Quality Practice v.1.0.0.1 and related documents. The particular software-development life cycle being used for this release is the Spiral Model, cf. Software Development Procedures 1.2.1. section 6.

5.2           Team and Roles

The tasks and activities associated with Quality Assurance are described in the Test Strategy document

6.0     Tools and Training

The staff for this project has been reviewed and the following gaps were identified: lack of quality process training for some staff members; lack of documented evidence of quality training. This was remedied by a training class held, and recorded, on October 24, 2007.

The Standard Development and QA tools will be used.
