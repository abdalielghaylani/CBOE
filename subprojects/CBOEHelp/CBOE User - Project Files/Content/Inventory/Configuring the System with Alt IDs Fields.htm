Configuring the System with Alt-Ids Fields

Alt-Ids fields may be configured in the invconfig.ini file. The user should have administrative privileges in order to edit the invconfig.ini file. Alt-Ids fields allow the user to enter additional information about a specific item.

The following example shows how to configure the system with Alt-Ids on the substance table:

  1. On the ChemOfficeEnterprise server, go to C:\Inetpub\wwwroot\ChemOffice\ChemInv\config folder
  2. Open invconfig.ini file in edit mode
  3. Modify the [SUBSTANCES] section as shown below:

[SUBSTANCES]

GET_CARTRIDGE_MW_FORMULA="False"

DISPLAY_SAFETY_DATA=TRUE

/////////////////////////////////////////////////////////////////

// Compound Alt Ids format: Field_Name:Field Label;Required-U. The -U indicates Unique identifier

// Example:

// ALT_IDS = inv_compounds.ALT_ID_1:Substance ID
1;1-U,inv_compounds.ALT_ID_2:Substance ID
2;0,inv_compounds.ALT_ID_3:Substance ID
3;0,inv_compounds.ALT_ID_4:Substance ID
4;0,inv_compounds.ALT_ID_5:Substance ID 5;0

// ALT_IDS=

ALT_IDS=inv_compounds.ALT_ID_1:Hazard;0,inv_compounds.ALT_ID_2:Precaution;0

So that ALT_ID_1 is being used to store hazards and ALT_ID_2 field is being used to store precautions (both are strings).

Alt IDs should be defined in the following format:

<Field_Name>:<Field_Label>;<0/1>

where:

  • Field_Name is the name of the alt ID field (as shown in the ini file)
  • Field_Label is how the alt ID should show up in the interface
  • The last parameter indicates if the field is required (1) or not (0)
  • Add a -U at the end of an alt ID's definition to make the ID a unique identifier

Note:you have to use the table name as a prefix for the field name while defining the Alt_ID.
