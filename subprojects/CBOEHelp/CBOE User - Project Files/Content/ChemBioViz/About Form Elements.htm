About Form Elements

Form elements are objects, such as text boxes, tables, or controls, that you use to build forms. Elements can hold text, structures, chemical formulae, and pictures. You can even add button elements to your form to invoke commands. After you build a form and you use it in a query, the query results populate the form and the data appears in the form elements. By choosing form elements and how to arrange them, you can view the data any way you want.

The elements you can add to a form are briefly described below. For more information, please refer to "Form element types".

Textbox

Use a text box to display (or allow users to enter) plain text and numbers.

Button

Use buttons to enable users to launch applications, open a Web browser, or perform other actions.

RichtTextBox

Use a rich text box to display formatted text and numbers. Formatting can include bullets, text styles, and active Web addresses.

ChemDrawBox

Use the ChemDrawBox to enable users to draw query structures and reactions. When a user clicks the box in the form, the ChemDraw toolbar appears.

FormulaBox

Unlike text boxes, formula boxes are specifically designed to display chemical formulae in their proper format.

PictureBox

Use a picture box to display a logo or other image in your form. Form users cannot modify PictureBox content.

MonthCalendar

If your data includes dates, use the MonthCalendar element to display the dates in a calendar rather than a text box.

DateTimePicker

This element displays the current day and date. Users can also select a day and date if the dataview includes this data.

SubformGrid

Although forms are designed to display data in a custom format, you may still want to display some data in a table. In this case, you use a subform grid.

ComboBox

Use a combobox to enable users to select a query parameter from a list of options.
