When you create a button, you assign three basic properties to it:

  • The argument (a string of text or data) on which the button performs the action
  • The label that appears on the button
  • The action that the button performs

Adding a button to a Form

To add a button:

  1. In the form editor, click+drag the Button icon from the Toolbox onto the form.
  2. Select the button.
  3. In the Properties window, select (Advanced) in the Misc. section.
  4. Click on the browsing button.The CBV Button Properties dialog box opens.
  5. Select the action from the drop-down list.
  6. In the Button Label text box, enter the label to appear on the button (such as Go to PerkinElmer.com).
  7. In the Argument text box, enter the action to perform when the user clicks the button.
  8. (Optional) To add a variable to either the argument or label
  1. Select from one of the Variable drop-down lists:
  • Select from Data field to add a database field variable.
  • Select from Variable to add a system variable.
  1. Place the cursor in the Argument text box where you want the variable to appear.
  2. Click Insert.
  1. Click OK.

Assigning Properties to a Button

  • Assigning an Argument

The argument provides information needed to perform the action. For example, to create a button that launches a Web site, the argument would include the URL; alternatively, a button that displays a popup message would contain the message text or a variable that represents data to display.

  • About variables

A button argument can have one or more variables. A variable represents data that ChemBioViz uses to execute the button action. For example, to enable users to search the Internet for the MSDS of the chemical product currently displayed in the form, you might use this argument:

MSDS %PRODUCT:PRODNAME%

where MSDS is ordinary text and the variable%PRODUCT:PRODNAME% is a field name in the form that represents the product currently displayed. Variables may be either system data or a field from the current table.

There are several system data variables:

AuthenticationTicket

An encrypted string that contains login data for passing to other ChemBioOffice Enterprise applications

UserName

The name of the logged-in user

DatabaseName

The name of the current data view

TableName

The name of the base table connected to the current form

FormName

The name or local path of the current form.

TodaysDate

The date in culture-dependent “long date string” format

AppDataPath

The path to the ChemBioViz data folder, that contains settings and user files

StartupPath

The path where ChemBioViz exe is located

  • Assigning Labels

A button label can include text and variables. To change the font, select the desired options in the Appearance section of the Properties window.

  • Assigning Actions

Only one action at a time can be assigned to a button.The actions you can assign are:

None

No action on click. (Even when no action is specified, a button click generates an event which can be caught by an external application.)

Launch Program

Launch a software application. The argument is the program name or path and can include a parameter list. You must provide the program’s full path. For example, to open the file filename using the program Application, the argument would be: “C:\Application.exe” “C:\filename.txt”.

Launch URL

Launch a Web site in a browser window. The argument is the full URL.

Launch Document

Open a document in its native application. The argument must be the full path name to the file.

Open Form

Open a ChemBioViz form. The argument is the name of public or private form or the path to a local form file. Include the path if the form you want to open has the same name as another form.

Google

Perform an Internet search using Google. The argument is the search query (may not work on all browsers).

Message

Display a message in an alert box. Argument is the message to be displayed.

Change Tab

Bring a specified tab to the front. The argument is the tab name.
