Merging Queries

Merging is a Boolean method for combining two hitlists to create a new hitlist. For example, you can run a substructure query for phenol, another query for hydroxyl compounds, and then remove one set of results from the other. Alternatively, you can add two hitlists together (duplicates are removed in the merged list).

A merged query cannot be rerun. However, you can delete and recreate merged queries as desired.

To merge queries:

  1. Perform the two queries you want to merge.
  2. Open the Queries pane. Both queries appear in the Queries pane.
  3. In the queries pane list, do one of the following:
  • Click+drag one query onto another.
  • Right+click one of the queries and choose Select for Merge; right-click the other query and select Merge with.

  1. The Merge Lists dialog box appears. A label is created for each query: L1 and L2.
  2. In the Merge Lists dialog box, indicate how you want to merge the two queries:

Intersect

Reports the results that meet the criteria of L1 and L2.

Subtract

Removes the results of L2 from L1.

Union

Reports results that meet the criteria of either L1 or L2.

Subtract from

Removes the results of L1 from L2.

Another method for merging queries is to assign one query as a target and merge other queries with it:

  1. In the queries pane, right-click one of the listed queries and select "Select for merge" from the context menu.
  2. Right-click another query in the list and select Merge with {the target query}.
