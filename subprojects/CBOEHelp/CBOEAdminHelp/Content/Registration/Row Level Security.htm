Row Level Security

The Registration system can be optionally configured to implement project based virtual databases. This option known as “Row Level Security” (RLS) prevents certain users from accessing individual database rows based on their security clearance. Row level security is applied based on associations between users, projects, and registered compounds. A user will only see those registered compounds that are associated with projects to which he/she has been granted permissions.

RLS is the visibility control of the data based on projects:

  • RLS-Batch: If the user is not assigned to a project, he/she is not able to see the particular batch data
  • RLS-Registry: If the user is not assigned to a project, he/she is not able to see the entire registry record data

Edit or delete a record/batch is not allowed to users if they don’t have full visibility for the record. The only way to get around the rule is by giving the user the special administrator privilege: RLS Exempt privilege

Edit Scope is the editability control based on the key privileges: EDIT_SCOPE_SUPERVISOR and EDIT_SCOPE_ALL.

Note that RLS is dominating edit scope privileges. Even if the user has edit scope all privilege, he/she is not allowed to edit/delete the record/batch if he/she doesn’t have full visibility for the record.

The Row Level security feature can be enabled/disabled using the configuration tool dialog during the installation.

The enable/disable the RLS:

  1. Go To Start>>All Programs>>ChemBioOffice Enterprise>>Server Configuration Tool. The Configuration Tool dialog displays. Select "App Settings" tab as shown in the screenshot below:

  1. From this tab, it is possible to Enable/Disable the RLS feature.
  2. To disable the feature, select the "RLS OFF" radio button, then users will be allowed to access whatever projects are available in the Projects table.
  3. When the RLS feature is set to true, then users can access projects to which they have specific access privileges or can access all public project.
    To enable the RLS feature. Two options are available:
  • Select RLS at Registry Level radion button, then the Project dropdown list will be available in the Registry Information section within the Submit Record page.
  • Select RLS at Batch Level, then the Project dropdown list will be available in the Batch Information section within the Submit Record page.

RLS Exempt user

If a Test user having admin privileges creates PRJ01 project as private project, then only Test user can access that project.

In general, users can access their private projects and all public projects.

The RLS Exempt user can access all projects (Private and Public) and their associated records. If any user has RLS_Exempt privilege checked then if we consider the example above, he/she can access the PRJ01 project.

Batch level security

The Batch level security feature allows the user to edit his/her own batches, but not the registry record as a whole when RLS is set at the Batch Level. Please refer to the following example for more information:

User A

  • Creates a record A
  • Assigns a private and a public projects at the batch level

User B

  • Is allowed to see the record A
  • Is not allowed to edit the batch A
  • Is allowed to add new batches to the record A
  • Is only allowed to edit his/her newly added batches

If RLS is set at the Registry level, unauthorized users can add new batches but they cannot edit existing and even their newly added batches. Please refer to the following example for more information:

User C

  • Creates a record C
  • Assigns a private and a public projects at the Registry level

User D

  • Is allowed to see the record C
  • Is not allowed to edit the batch C
  • Is allowed to add new batches to the record C
  • Is not allowed to edit his/her newly added batches
