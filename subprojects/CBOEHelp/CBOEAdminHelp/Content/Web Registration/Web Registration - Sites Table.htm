Web Registration - Sites Table

Sites refer to different locations from where users can access the Registration application and submit / register records to the temporary / permanent database.

Let us say that a company has two branches, one in Boston and the other in London. Both branches are using the same product pointing to same server. The admin will do the following:

  • Create two sites BOS for Boston and LON for London
  • Create two sequences BO and LN
  • Assign BOS site to BO sequence and LON site to LN sequence

Every record registered in Boston will have BO as prefix and every record registered in London will have LN as prefix. This is an implicit visual differentiation for finding the source of a registered record.

Adding a new site:

  1. In the Web Registration Admin window, select Sites from the Custom Tables drop-down list. The Sites window appears.
  1. Click on the plus icon, a new form pops up as shown below.

  1. Enter the Site Id in the Sites Id field. The Site Id is used to give a site of identifier for the site
  2. Enter the Site Name in the Sites Name field. The Site Names are displayed in the Site drop-down list
  3. Click on Save button.
