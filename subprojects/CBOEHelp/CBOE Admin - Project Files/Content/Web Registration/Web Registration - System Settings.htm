Web Registration - System Settings

The administrator user has the privilege to manage the system settings and configuration in order to specify the Registration's workflow.

Multiple features may be enabled/disabled from the Registration system settings.

To open the Registration system settings:

  1. Log in Web Registration with administrative privileges
  2. Click on System Settings from the System Behaviors drop-down list.

The following window appears:

  1. To enable a feature, the administrator has to set its value to True and to disable a feature, its value should be set to False in the table.

Below are some of the Registration System Settings:

  • ApprovalsEnabled: If set to true, it enables the approval workflow, including a new level to the registration workflow – the Reviewer. The reviewer is responsible for checking the submitted records and approving them. Only approved records can be registered. The workflow, and the ability to approve/remove approval of a submitted record are controlled by both a system level setting, ApprovalsEnabled, as well as user privileges set_approval_flag and toggle_approval_flag.

    If ApprovalsEnabled it is set to False, the registration workflow will only contain two levels, Submitter and Registrar.
  • SameBatchesIdentity: If it is set to True, allows different options to appear during the duplicate resolution workflow and determines whether or not batches can be added to existing records or if unique records are created instead.
  • CheckDuplication: If it is set to True, enables validation of duplication of structures and mixtures with identical compound before of the Registration.
  • EnableMixtures: If it is set to True, enables managing more than one compound in a registry record. The submission of new components and the submission of mixtures are separate.

    With the above settings, in the Main page of registration two icons will be available for the submit workflow: Submit New Component and a Submit New Mixture. Note that the user will not see the Submit New Mixture icon on the CBOEManager home page.
  • AllowUnregisteredComponentsInMixtures: If set to True, allows components to be added to mixtures without being registered.
  • AllowCreateCopySharedStructure: If set to True, allows to create copies of the edited linked structures while editing a mixture created from registered components.

  • EnableSubmissionTemplates: If set to True, enables the ability to save and load submission data prior to actual submission or registration.
  • EnableSubmissionDuplicateChecking: If set to True, enables duplicate checking to occur when records are submitted and requires suggested action from submitter. Additionally the user can add additional comments. This information is displayed in the detail view of review/register screen. In List view if there were dups found on submission a “dups found” text will appear under the review link.
  • EnableFragments: If set to True, enables the use of fragments to track salts and solvates.
  • EnableDuplicateButton: If set to True, enables the display of Duplicate button during component duplicate resolution.
  • EnableAuditing: If set to True, it will enable registration triggers in Oracle to create audit in Registration actions. This feature enables the audit of all actions on records in the following permanent registry tables: compound_molecule, structure, batches and mixtures has been added. Changes are added to the Auditing_History table in the regdb schema.
  • EnableAddBatchButton: If set to True, enables the display of Add Batch button during component duplicate resolution
  • EnableUseBatchPrefix: If set to true, it enables a prefix at the batch level instead of the prefix at the registry level. Only one should be available.
  • DefaultBatchPrefix: This is to enter the batchprefixID user wants to use while uploading records in bulk registration (used for DataLoader and InvLoader)
  • BatchHighlightingValue, BatchHighlightingTooltip and BatchHighlightingField: If values and entered, enables to highlight batches if the configured field has the configured value.

  • MarkedHitsMax: This setting is the maximum number of marked hits allowed by a user during a session. The default value is 500, but it is recommended to configure MarkedHitsMax as 200.
