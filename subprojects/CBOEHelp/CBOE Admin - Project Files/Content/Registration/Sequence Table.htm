Sequences Table

Sequence tables store prefixes and starting numbers for the Registration numbers of the compounds or components.

This information appears in the Prefix dropdown list, when the compounds are added to the Registration Enterprise.

A Registry Number is defined by these properties:

  • Registration number padded length: Total length of the Registry Number. The Registration System will 0-pad Registry Numbers to the total length, so all Registry Numbers have the same length.
  • PREFIX: RegNumber Prefix
  • PREFIXDELIMITER: Delimiter between the prefix and the Registry Number
  • SUFFIXDELIMITER*: Delimiter between the Registry Number and the SALT name
  • Salt Suffix Type*: Fragment Code (Salts/Solvates)
  • BATCHDELIMITER: Delimiter between the RegNumber and the batch number on the full reg number field.
  • Batch number padded length: Total length of the Batch Number.

* Only when registration system has EnableFragments=True and SameBatchesIdentify=False settings

Entering a New Sequence

  1. In the table editor page, select the "SEQUENCES" table and then click on Add button

  1. Fill in the sequence's properties and then click on Add button.

  1. The new sequence appears in the Sequences Table.

Sort Order feature

The sort order feature allows the administrator to specify the order of appearance of a picklist's values. Note that the top most value will be selected as default value. In the prefixes example, the prefix that has "1" as Sort Order value will be the default value in the Prefix field within the Submit Component window.
