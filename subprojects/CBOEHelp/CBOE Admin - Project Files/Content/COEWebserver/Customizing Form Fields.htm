Customizing WebServer Application Forms

WebServer applications created using the Administrative Tools Wizard, use the default options to display form fields.  These fields can appear in many different ways. The showinputfield and ShowResult functions in the form ASP pages indicate how a field should be displayed. 

This is not valid for Registration System or Inventory Manager, which use alternate ways to format the forms.  Please see the application's Admin Guide for more details.

It is important to note that it is the responsibility of a site's administrator to propagate any changes made to ChemOffice WebServer application files to future versions of the application.  New versions of the ChemOffice WebServer use default configurations.

ShowInputField and ShowResult

ShowInputField (for search forms) and ShowResult (for result forms) are used in each application form ASP pages to define how a field should be displayed.  To find the appropriate field entry:

  1. Open the ASP page defining the form you want to change.
  2. Find the line starting with %ShowInputField (for search forms) or %ShowResult (for result forms) and includes the field name as one of the inputs.

Syntax

The syntax for any field entry is:
%ShowInputField dbkey,formgroup,TableName.<FieldName>,<Input Field>,<Option> [; <Display Field> : <Option>]
The Display Field option is optional.  So, the syntax for a field with no Display Field Option (defaults to TEXT) is:
%ShowInputField dbkey,formgroup,TableName.<FieldName>,<Input Field>,<Option>

Each Input Field types have their own Display Field options.  Please see below for more details about each type of field.

Please do not change the Fieldnames or add or remove any of the items in the list. If you see NOT CONFIGURABLE next to the field this means the field is not configurable.

For more information about another parameter in this statement, please see the documentation for ShowInputField or ShowResult .

ShowInputField Options

The ShowInputFields function allows the following types of fields:

  • TEXT
  • CHECKBOX
  • TEXTAREA
  • PICKLIST

ShowResult Options

The ShowResults function allows the following types of fields:

  • TEXT
  • CHECKBOX 
  • HYPERLINK
  • PICKLIST (only seen this way in Edit mode)
  • FORMULA (formatted with subscripted numbers)

TEXT Options

TEXT fields can have a validate option associated with it.  Validate options indicate that after a user enters something in the field, the application checks to see if what was entered is in the correct format. 

The following options are available:

Option Description
1 Valid Integer
2 Float
3 Positive Integer
5 Positive Float
8 Date in the form: mm/dd/yyyy
21 CAS Number
22 not empty
24 Formula

To make a default value appear in the textbox, add an exclamation point (!) followed by the value to the field definition. 

CHECKBOX Options

The following are the two options for a Checkbox field:

Option Description
0 Shows Raw Value
GIF Shows a GIF with an X for values of "" or 0 and a CHECK for values of 1

HYPERLINK Options

The format for Hyperlinks is HYPERLINK:Link_Type|Link_Name. 

  • Link_Type is the type of the link (e.g. http, mailto, file, https, ftp)
  • Link_Name is the name that should be displayed for the link.  If this is left blank, the full link is displayed.

The following shortcuts can also be used instead of the Link_Type|Link_Name format:

Option Description
LINK displays the working link as the link text. Adds http:// if missing
FULL displays the full value in the field as link text. Add http:// if missing

PICKLIST Options

The picklist option consists of a list of items you want in the listbox.  Each item is separated by a pipe (|).  The default value (i.e. the value displayed in the listbox by default) is found at the end of the list after an exclamation point (!).

For example, if the picklist is a field called Names with the following options: Jane, John, and Bill where John is the default value, the field definition would look like this:

"Table.Names"; "Names"; PICKLIST:Jane|Bill!John
