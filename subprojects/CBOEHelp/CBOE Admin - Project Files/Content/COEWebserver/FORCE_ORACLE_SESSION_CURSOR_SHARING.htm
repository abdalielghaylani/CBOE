FORCE_ORACLE_SESSION_CURSOR_SHARING

Example: FORCE_ORACLE_SESSION_CURSOR_SHARING=1

Description: Indicates if cursor charing is used during searching and large data loads. Although the default is zero, in general, this should be turned on (set to 1) in order to improve performance. All versions of Oracle 9i can utilize this feature, but only some versions of Oracle 8i show improvement with this set to 1 (e.g. Windows 8i version shows no improvement so this flag should be set to 0). In general, if you are using Oracle 9i, set this value to 1. If you are using Oracle 8i, set this to 0.

Default: 0

Options: 0 | 1

It is important to note that it is the responsibility of a site's administrator to propagate any changes made to ChemOffice WebServer application files to future versions of the application. New versions of the ChemOffice WebServer use default configurations.
