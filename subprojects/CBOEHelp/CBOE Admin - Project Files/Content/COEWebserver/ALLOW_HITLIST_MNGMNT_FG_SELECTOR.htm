ALLOW_HITLIST_MNGMNT_FG_SELECTOR

Example: ALLOW_HITLIST_MNGMNT_FG_SELECTOR= 1

Description: This is only relavent is cases of a global search. Using hitlist management you can recover and operate on lists that were saved under a given form group. That is, hitlists are formgroup specific so the hitlist dialogs show lists from the currently open formgroup. The exception is global search applications where a global search form deals with multiple form groups simulataneously. If ALLOW_HITLIST_MNGMNT_FG_SELECTOR is set to 1, an additional dropdown in the hitlist management dialog appears to allow you to select which histlist you want to save (the hitlist from which formgroup).

Default: 1

Options: 0 | 1

It is important to note that it is the responsibility of a site's administrator to propagate any changes made to ChemOffice WebServer application files to future versions of the application. New versions of the ChemOffice WebServer use default configurations.
