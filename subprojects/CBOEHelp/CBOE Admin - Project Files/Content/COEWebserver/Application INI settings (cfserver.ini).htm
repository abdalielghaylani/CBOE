Application INI settings (cfserver.ini)

The cfserver.ini file is the application ini file. It specifies the DataViews within the application as well as some application variables. The file defines parameters that allow the ChemOffice WebServer application to find necessary information and how to react in certain situations.

There is a cfserver.ini for each application located in: <webroot>/ChemOffice/<app_name>/config where <webroot> is your server's webroot and <app_name> in the application name.

When an application is started from a closed state, which occurs when a request for any page within the application directory occurs, the global.asa application level functions and subroutines are run. These functions and subroutines read the ini files and store information in application objects regarding general information about how the application should function. These variables are created only when the application is started, and will remain constant throughout the life of the application unless, when a new user enters a page, a change is detected in one of the config files, at which time the application will reload the new variables.

When editing fields in this file, it is important to be sure what is going to change because many of these settings are necessary for the application to function properly.

It is important to note that it is the responsibility of a site's administrator to propagate any changes made to ChemOffice WebServer application files to future versions of the application. New versions of the ChemOffice WebServer use default configurations.

The following is a list of entries that are usually found in the cfserver.ini files for ChemOffice WebServer applications. Click on an item in the list for more information.

If these entries are missing from the cfserver.ini file that you are trying to customize, the default value is used. Adding the entry to the cfserver.ini file allows the administrator to explicitly specify the value.
