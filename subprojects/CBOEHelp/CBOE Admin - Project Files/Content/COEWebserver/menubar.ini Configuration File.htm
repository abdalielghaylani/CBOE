Menubar.ini Configuration File

The menubar.ini configuration file (found in the config folder of each application's folder) controls how the menubar is displayed in the application.  By default, the following menus are available to the user:

  • File
  • History
  • Queries
  • Hitlists
  • Marked Hits
  • Help
  • Logoff
  • Home

Menubar items are characterized as a menu, submenu, or button (an item with no menu items). System administrators are able to add new menus to the above list of defaults.  Each menu is allowed to contain up to 3 menu items, and one of those 3 can contain a submenu.  There is a maximum of 2 items in each submenu. 

It is important to note that it is the responsibility of a site's administrator to propagate any changes made to ChemOffice WebServer application files to future versions of the application.  New versions of the ChemOffice WebServer use default configurations.

Menu Definitions

There should be a menu definition in the menubar.ini file for each menu listed in  MENUBAR_ITEMS. If there is no definition, please see your system administrator.

Menu definitions consist of a number of settings describing the menu as well as item definitions for each item in the menu (including items which happen to be submenus.  Menu definitions always begin with a section heading which is the name of the menu (as listed inMENUBAR_ITEMS) in brackets.  For example, the following is the beginning of a menu definition for the FILE menu:

[FILE]
MENU_TYPE = Menu
MENU_HIDE=
MENU_FORMGROUPS=
MENU_NAME = File

NUM_MENU_ITEMS=3

Each Menu definition must contain one and only one entry for each of the settings listed above. 

Item Definitions

Each item in a menu must have an item definition listed under the appropriate menu section heading in the menubar.ini file.  If there is no definition, please see your system administrator.

Item definitions consist of a number of settings listed as the settings name, followed by an underscore (_), and finally followed by a number.  All of the settings for a particular item end with the same number.  For example, the following is a definition for the Preferences Item under the File Menu:

NAME_ITEM_1=Preferences...
NAME_ADD_INFO_ITEM_1=
URL_ITEM_1=
ONCLICK_ITEM_1=MainWindow.getAction('preferences')
DIM_ITEM_1=
HIDE_ITEM_1=
FORMGROUPS_ITEM_1=
HELP_ITEM_1=open preferences window to set search and display options
ICON_IMAGE_ITEM_1=

Item definitions must contian one and only one entry for each of the settings listed above .
