Loading Substances

A substance is a material with a specific chemical composition. A substance can be either a pure chemical element or a pure chemical compound.

Data Loader lets you load substances in the following ways:

  • Pending Review: Lets you load substances into the temporary table of Registration Enterprise. These substances need to be reviewed before being registered, and hence they belong to the “Pending Review” list.
  • Registered: Lets you load substances into both temporary table and permanent table of Registration Enterprise. Depending on the circumstances and Duplicate Action settings, some substances will go into temporary table instead of permanent table. For example, when you select the Add to substances pending review option, duplicate records go to the temporary table and non-duplicate records go to permanent table. The other options allow you to add a substance as a duplicate registered substance or a new batch of a registered substance.
  • Register with known RegNum values: Lets you load substances into Registration Enterprise by specifying the registration number instead of having it automatically generated. You can use this option if you are having external databases that point into their Registration system. Duplicate Action settings are also available with this option.
  • Check for Registered Duplicates: this option enables the ‘matching-only’ mode after choosing ‘Load to Registration’, where the choices of registration type and duplicate resolution are presented. Please note that the option is only available for the ‘permanent’ registration workflow.

Note: when Check for Registered Duplicates option is selected, then after the usual data-mapping and range-selection screens, the matching is performed as per the System Settings chosen, and results are displayed. When a compound from the data-file has been matched with an existing registration, the results indicate both the matching mechanism and provide a comma-delimited list of Registered compounds that were matched. Matched compounds are conveniently marked as ‘errors’ to allow the user to easily filter the results using the ‘View only errors’ checkbox.

To load substances into Registration Enterprise:

  1. Log into Data Loader with appropriate credentials to load data into Registration Enterprise.
  2. Select Load substances to registration option.
  3. Click Next. The following screen appears as:

  1. Select the desired option and click Next. This screen appears as:

  1. Click Browse and select the file to import. For information on the file formats that you can import, see “Importing a data file to load”.

NOTE: If the selected input file is a .cfw or .mdb file with multiple tables, or a .xls file with multiple sheets, then you must choose the table or sheet you want to import.

  1. Click Next. The following screen appears and shows input fields, data types, and values, as shown below:

  1. Select the appropriate data type corresponding to each input field. You may also change the labels of the input fields or sort the fields as desired. For more information, see “Sorting data”.

NOTE: By changing the settings, you can control how the input data is interpreted. However, the input file itself is not modified.

NOTE: Use the Save button to save the settings you select or the Load button to load the settings that have already been saved.

  1. Click Next. This screen appears as:

This screen shows the input fields, output fields, and their corresponding data types. The output fields are classified into four sections, which are:

  • Registration Information
  • Component Information
  • Fragment Information
  • Batch Information

The above classification is based on the way records are submitted into Registration Enterprise. For more information on the output fields belonging to these sections, see “Output fields”.

The list of output fields and output field types depends on the data management task that you have chosen. You cannot edit the output field names or data types.

  1. Map output fields to input fields. You can specify how a particular output field will be populated. Depending on the output fields, these options are available in the dropdown lists of the Value From column:
  • Default: Indicates that the output field value is optional and is supplied by the task.
  • Input field: Lets you map an input field. Only input fields of a compatible data type may be mapped.
  • Picklist: Lets you choose a value from a list.
  • Constant: Lets you enter a constant value.
  • Date: Lets you enter date. On clicking the down arrow next to the date, a standard date picker appears.
  • Calculation: Lets you map an output file with the result of a calculation. A dropdown list appears under the Value column and displays a small box with ellipsis. The color of this box indicates whether the formula is valid; green is for valid and red is for invalid. On clicking the ellipsis inside this box, the Calculation window appears as:

Click the Functions button to choose calculation function. For more information on the available functions, see “Functions”.

Click the Fields button to choose the field to be passed to the function.

On selecting Input field from any of the dropdown lists under the Value From column, a drop-down list appears under the Value column, corresponding to the field. This dropdown list contains all the input fields, which are compatible with the data type of the Output field.

  1. Click Next. The job configuration screen appears as:

  1. Configure the job. To configure the job, specify the record from where processing should start, specify the number of records to process, and select whether the job should start immediately or later.
  2. (Optional) Save the job if you would need to execute it later.
  3. Click Next. The loading process window displays as shown below:

  1. When the substance data is loaded into Registration Enterprise, the following Job summary appears as:

In the above screen, messages are displayed in the form of grid with two columns: Index and Message. The Index column lists the physical record index in the source file while the Message column displays the text of success message or abbreviated version of error encountered.

The View only Errors check box allows you to view only the log entries representing failed record inputs from the source file. Double clicking any error related row displays a printed version of the XML file that the Registration System created and tried to upload.
