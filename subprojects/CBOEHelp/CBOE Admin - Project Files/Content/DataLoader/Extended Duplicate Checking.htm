Extended Duplicate Checking

This step is only performed if the "Find Custom Field Duplicates" addin is enabled. The step involves a query matching a designated incoming property value against the contents of a corresponding column in a database. The query contains a filter of the form UPPER(<COLUMNNAME>) = UPPER('<PROPERTYVALUE>'). It is therefore critical that the designated database column is indexed using a unique functional index on the UPPER column value.

The table and column involved in the duplicate check query depends on the settings specified in the "Extended Duplicate Check" tab under Customize System Settings dialog.

  • If type is set to "Property" and Level is set to "Structure" the table to index is REGDB.STRUCTURES
  • If type is set to "Property" and Level is set to "Component" the table to index is REGDB.COMPOUND_MOLECULE

The name of the column to index will be determined by the property name indicated in the "Value" setting.

The index should be created using the following syntax:

CREATE UNIQUE INDEX <INDEXNAME> ON <TABLENAME>(UPPER(<FIELDNAME>));

This index should be created at the time when the "Find Custom Field Duplicates" addin is enabled and should not be dropped, unless the addin is disabled.

  • If type is set to "Identifier" then there is no need to add a custom index. A proper functional index alreday exists in the identifiers tables.
