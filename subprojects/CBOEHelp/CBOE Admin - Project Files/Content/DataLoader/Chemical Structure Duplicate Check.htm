Chemical Structure Duplicate Check

This is the key step required to ensure that registered records are chemically de-duplicated. The step involves a query matching the incoming chemical structure against the contents of the REGDB.STRUCTURES.STRUCTURE database column. The query is performed by the CSCartridge. It is critical that a property maintained CSCartridge Index exist on the above column. This step is expected to get gradually slower as the number of records present in the database increases. The time required to perform the structure match is dependent on the duplicate check parameters configured in the Registration System. The duplicate search settings determine whether factors such as stereochemistry, charge, isotopic differences, and tautomerism should be examined in determining potential chemical matches. The default duplicate check settings for the web based Registration application are fairly broad. They may typically flag several existing database records (different stereoisomer, isotopically substituted variation, differently charged structures), which the Registrar will visually inspect in order to choose the most optimal match. These broad duplicate check settings will result in longer query times. For bulk loading scenarios, where the data has already been de-duplicated in the originating system, and where there is no manual intervention by the Registrar to review the potential duplicates, it is better to use narrower match parameters that will result in faster and more targeted matches. In particular, enabling the IDENTITY search parameter will result in the fastest possible load performance.

The duplicate check settings can be adjusted via the Customize Registration > System Settings > Duplicate Checking Tab in the Registration Web Application.

Recommendations:

Before Bulk Load:

  • Set System Setting> Duplicate Checking >
    • IDENTITY = YES
    • TAUTOMER = NO
    • RELATIVETETSTEREO = NO

After Bulk Load:

  • Return Duplicate Check settings to pre-load state
