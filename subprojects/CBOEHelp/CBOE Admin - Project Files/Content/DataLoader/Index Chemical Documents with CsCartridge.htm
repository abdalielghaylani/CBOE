Index Chemical Documents with CsCartridge

There are four CSCartridge indexes on the REGDB database. Two of the indexes are only used for application search screens. One is only populated when data is inserted into the fragments table. The last one is used for duplicate check searching when records are inserted in the database.

The CSCartridge indexing process is CPU and disk IO intensive. Normally the indexes are automatically updated row-by-row as data is inserted. Data inserted into the REGDB.MIXTURE. STRUCTUREAGGREGATION column is not used for duplicate checking. During bulk loading operations it is therefore beneficial to temporarily suspend the REGDB.MX3 index. The index can be synchronized after the load is completed.

The REGDB.MX index on the REGDB.STRUCTURES.STRUCTURE column is very important for duplicate matching during data load. It should remain enabled.

Recommendations

Before Bulk Load:

  • Disable the MX3 Cartridge Index

connect regdb/oracle@<TNSName>
alter index regdb.mx3 parameters('DEFER');

After Bulk Load:

  • Re-enable the MX3 Cartridge Index

connect regdb/oracle@<TNSName>
alter index regdb.mx3 parameters('SYNCHRONIZE');
