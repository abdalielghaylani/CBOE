Adding Data

There are two methods for adding data to the database of Registration Enterprise. You can either load the data to the temporary table where it can be reviewed before it is added to the permanent table; or, you can submit the data directly to the permanent table without the data being reviewed. Regardless of the method you choose, you first import the data source file and map its data fields and data types to match those that already exist in the database. You then configure the 'job' settings to upload the data.

A job is a set of control data that directs Data Loader to perform a specific task. Data Loader lets you construct and configure jobs so that you can execute or modify them later. The most common job of Data Loader is to import singular or bulk data from a file into an Oracle database, text file, or XML file.

Note: Registration Enterprise uses an Oracle database for storing and managing data.
