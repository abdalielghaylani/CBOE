Project Table

The Projects table stores the projects that can be associated with registry records or batches. This information appears in the Project dropdown list when compounds or batches are added.

Entering a New Project

To enter a new project:

  1. In the table editor page, select the Projects table and then click on Add button

  1. Enter the Project name in the Name text box
  2. Enter the Project description in the Description text box
  3. Select Yes from the Is Active? list box if you want the new project to appear in the Project dropdown list.
  4. Select the Type of the project, specifying at which level the Project field will be displayed, Batch level, Registry level or both.

  1. The new project will be added to the Project's Table.

Note that when Row Level security is enabled, a new field is added to the Table Editor: Is Public?. If you select Yes from the dropdown list , the project becomes public and accessible by all users. If you select No, the project becomes private and accessible by the owner and a number of user to whom you provide the access privilege.

When RLS is set at the Registry level or at the Batch level, the Type field is hidden and then you cannot add/edit the project's type.
