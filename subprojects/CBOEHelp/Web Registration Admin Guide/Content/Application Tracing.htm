Enabling Application Tracing

You can configure CBOE applications to display trace information. Tracing provides timing information for the tasks performed in the application being traced. It helps in debugging and can be used to pinpoint specific trouble areas.

Note: You cannot enable tracing for Registration and Data View Manager applications using Application Tracing option under Security. For more information on enabling tracing in Registration and Data View Manager, RLS Exempt privilege.

Note: To prevent exposing tracing information to the unwanted users, trace files should be deleted and the tracing should be disabled prior to placing the system into production. Further protection could be afforded by configuring file permissions on the file system folder where logs are written or by restricting IIS permissions on that folder. This issue only affects CBOE Classic ASP applications such as ChemInv and BioSARBrowser. It does not affect .NET based application such as Registration and CBOE Manager.

To enable application tracing:

  1. Click Application Tracing under Security in the CBOE home page.


  2. Select an application to trace from the list of applications available (refer figure below) and click OK.


  3. Select 'ON' in the Trace Status column. Click OK on the two pop-up windows that appear.


    Note: You can display trace information for individual pages by selecting 'Enable page timer' for your desired application.
  4. Minimize/close the window and perform the required tasks in the application.

To view the trace report:

  1. Click View in the View Trace column.


  2. The trace file showing timing information in seconds for the task performed appears. For example, see figure below:

To delete the trace report:

  1. Click Delete in the Delete Trace column.


  2. Click OK on the pop-up window that appears.

Enabling Tracing in Registration Enterprise and Data View Manager

To enable tracing in Registration and Data View Manager:

  1. Open the folder containing the web.config of the Registration/DVManager module. To open the folder, go to IIS (go to RUN and type the INETMGR command) and check the virtual directory assigned to COERegistration/ COEManager module.
  2. Open the web.config file and search for “coeLoggingConfiguration” node.
  3. Now activate the logging functionality by setting the “enabled” attribute value to “true” as shown below:


  4. Once tracing is enabled, the trace report can be found in the folder listed below on CBOE server: C:\ProgramData\PerkinElmer\ChemOfficeEnterprise\LogOutput\<client ip address>\UserName_<user name>_UserID_<user id>\SessionID_-1\
