CASE C - Working with Multiple Components

When the system has the following system settings:

  • SBI = N/A
  • EnableFragments = False
  • EnableMixtures = True

If the structure that the user wants to register is already in the Chemical Registration Database, a solving duplicate window will appear. The only field for checking duplicates is the structure one.

There are two levels of solving duplicates for Registry Records with multiple components:

  1. Solving Component Duplicates: For each component we can review a list of duplicates of the specific components. We can decide to use the same Component ID or to create a new Component.
  2. Solving Registry Record Duplicates: Once the components are solved we have to solve registry record duplicates. Only Records with exactly the same components list will appear.

Once the components are solved we have to solve registry record duplicates. Only Records with exactly the same components and fragment list will appear.

If the structure that the user wants to register is already in the Chemical Registration Database, a solving duplicate window will appear. The only field for checking duplicates is the structure one.

Many cases are provided here:

Note: in the following cases, Source is the record that the user is registering and Conflict is the record that is already registered in the database.

  1. Source = Single ; Conflict = Mixture

When the user is registering a component that is duplicated in a Mixture. The following duplicate resolution options will be available:

  • Cancel: to cancel the registration process
  • Duplicate: to create a new registry record duplicating the compound information. Then the record will be duplicated with the following properties:
    • Unique Reg Number
    • Different Component ID
    • Same Structure
    • Different Structure ID
  • Use Component: to use the same Component ID. No component is created.
  • Use Structure: to create a new Registry Record reusing the same structure ID. If there is a structure change both registry records will be affected.
  1. Source = Mixture ; Conflict = Mixture

When a user is registering a mixture containing two components, and the mixture is entirely duplicated, the following dialog displays:

The two duplicated components should be resolved one by one. Click on "Resolve this Component" link and select the action that you want to perform. The options are:

  • Cancel: to cancel the registration process
  • Duplicate: to create a new registry record duplicating the compound information. Then the record will be duplicated with the following properties:
    • Unique Reg Number
    • Different Component ID
    • Same Structure
    • Different Structure ID
  • Use Component: to use the same Component ID. No component is created.
  • Use Structure: to create a new Registry Record reusing the same structure ID. If there is a structure change both registry records will be affected

When the components are resolved, the Duplicate Registry dialog displays:

Select one of the following options to resolve the registry duplicate:

  • Cancel: to cancel the registration process
  • Add Batch: to add a batch to the currently displaying duplicated registry. This will conclude the duplicates resolution process.
  • Duplicate: to create a new registry record duplicating the compound information. Then the record will be duplicated with the following properties:
    • Unique Reg Number
    • Different Component ID
    • Same Structure
    • Different Structure ID
  1. Source: Mixture ; Conflict: Single

When the user is registering a mixture containing a component that is duplicated in a registered record having a single component, the following dialog displays.

Click on resolve this component, the following options shown in the screenshot below will be available:

  • Cancel: to cancel the registration process
  • Duplicate: to create a new registry record duplicating the compound information. Then the record will be duplicated with the following properties:
    • Unique Reg Number
    • Different Component ID
    • Same Structure
    • Different Structure ID
  • Use Component: to use the same Component ID. No component is created.
  • Use Structure: to create a new Registry Record reusing the same structure ID. If there is a structure change both registry records will be affected
