Viewing Contents of a Container

After searching for a container by navigating the Location Tree, or using a search form, you can view its contents using the following frames in Container Management:

  • Container List
  • Container Details

Container List

The Container List displays all containers and plates contained in the location selected in the Location Tree. The icons beside the containers lets you distinguish the containers from plates. When you click a container, the icon besides the container changes to the icon, and the details of the container display in the Container Details. See the following example.

The Container List consists of various columns with information about the key container attributes, such as, container name, location, and ID. You can click a column to sort the containers displayed in the Container List, based on the column. When you click a column, a icon is displayed next to the column name. This icon helps you determine the sorting order for the containers.

You can customize your view of the list of containers using one of the menu options listed below:

  • View: Click View to open a menu:
    • Multi-Select: Lets you select more than one container. After selecting multiple containers, you can perform an action on all containers, simultaneously.
    • Large Icons: Displays large icons corresponding to the list of containers.
    • Small Icons: Displays small icons corresponding to the list of containers.
    • Details: Displays the Container List in its default view.
    • Column Chooser: Lets you customize the appearance of the Container List. You can use Column Chooser to add or remove columns to, or from the Container List, and set the width of the columns. You can also use Column Chooser to specify the number of containers to display in a window of the Container List, at one time.

  • Rectify Contents: Verifies whether a container is at the location Inventory Enterprise records it at. Rectify Contents also lets you update Inventory Enterprise.
  • Print Report: Lets you generate reports containing information about the containers found in the selected location.

Note: Your Container List might not display all links described above. Your level of privilege determines which links you can access on your system. To determine if you have the appropriate privileges to access a link in the Container List, see Roles and Privileges.

Container Details provides detailed information about the container selected in the Container List. You can gather information about all the attributes of a container using Container Details. Since there is a large number of container attributes, similar types of container attributes are grouped under different tabs in Container Details: Summary, Substance, Supplier, Quantities, Comments, Reservations, Other, EH&S, and Requests.

For information about the fields in each tab, see Inventory Enterprise Form Fields.

Rectifying Contents

Sometimes, the location of a container in Inventory Enterprise changes, but Inventory Enterprise is not updated with the current change in location. This results in discrepancies in Inventory Enterprise. Rectify Contents lets you verify that a container's location is correctly referenced. Unlike the Update Contents function, the Rectify Contents function lets you correct discrepancies and update Inventory Enterprise with the correct information.

Rectify Contents should be used on quarterly, or biannual basis, during official reconciliation efforts. By rectifying content, you can:

  • Move containers not present at a location, but should be there.
  • Move containers present at a location, but should not be there.

To rectify contents of a location:

  1. Navigate to the location whose contents you want to rectify.
  2. Click Rectify Contents in the Container List. The Reconcile containers at this location window appears.

The lower frame of the Reconcile Location window contains the list of containers that should be found at the selected location.

  1. Type the barcode of the container you want to verify in the Scan Barcode text field.
  2. Press the <Tab> key. The check box corresponding to the scanned container is automatically selected. However, if the scanned container is not available at that location, the container is added to the list of the misplaced containers. Open the Misplaced tab to view the list of misplaced containers, which appears in the Reconcile Location window when you press the <Tab> key (after scanning a barcode). An example of misplaced containers is shown in the example below.

If the scanned barcode does not exist in the Inventory database, it is added to a temporary list of unknown/new containers. This list appears under the New tab and displays the unknown/new containers scanned/entered since the Reconcile Location form was first opened.

  1. Click Print in the upper-right corner to print this list. The application displays a printer-friendly container report that includes specific information about the location.

The temporary list remains within the Reconcile Form until it is closed, whether the report has been printed or not. Once you close the Reconcile Location form, the list is removed from memory and is no longer retained in any form within the application or database.

  1. Repeat the above steps for all other containers you need to verify location for.

Note: Uncheck the checkbox next to the misplaced container if you do not want to move it to the location you are rectifying.

  1. Click OK. If the list of the expected containers contains some containers not verified, the containers are marked as missing and a warning message appears.
  2. Click OK if you want to move the containers that have not been verified to the missing location.

Printing Reports

The Print Report link in the Container List lets you generate a report that provides summarized information about all the containers contained in the Container List. You can also generate barcodes for all the containers stored in the location using the Print Report link. A barcode is a special type of report that can provide information about any attribute associated with a container.

The reports are driven by the report layouts, which specify the format of a report and the information that is to be included in the report. Therefore, report layout that you choose while generating the report specifies whether to generate a report that provides information about all the containers contained in the Container List frame or to generate barcodes for the containers.

As reports are driven by report layouts, the reports may not provide the same information as provided by the Container List frame. For example, the Container List frame may display information about the costs of the containers, however, the report generated for the Container List frame may not provide this information.

Note: See your system administrator if the reports available in Inventory Enterprise do not meet your specific business needs.

To generate reports:

  1. From the Container List, click Print Report. The Reports window displays.
  2. Select the report layout from the Select a report layout drop-down list.
  3. Click Go. The Reports window populates with the Select a report format drop-down list.

  1. Select the report format from the Select a report format drop-down list.
  2. Click Go.

A warning message displays and informs you that the window, which will be displayed next possesses a security risk.

  1. Click Yes to continue. The results display in the lower frame of the Reports window.

Changing Qty (Quantity)

To confirm you have the appropriate privileges to change the quantity of the substance stored in a container, see Roles and Privileges.

The quantity of a substance stored in a container modifies when the substance is used in experiments. Change Qty lets you update Inventory Enterprise with the changes in the quantity of the substance.

To update Inventory Enterprise with the changes in the quantity of the substance stored in a container:

  1. Select the location in the Location Tree for the container you want to modify.
  2. Select the container in Container List. The details of the container display in the Container Details.
  3. From the Update menu in Container Details, select Change Qty.

The Change Amount Remaining in an Inventory Container window appears.

Note: For information about the fields in the Change Amount Remaining in an Inventory Container window, see Inventory Enterprise Form Fields.

  1. Type the quantity removed from the container in the Quantity Removed (mg) text field.
  2. Click OK. The value of the Quantity Remaining (mg) field is revised based on the value entered in the Quantity Removed (mg) text field.

Note:You can change the quantity by specifying a value in the Quantity Remaining text field, instead of specifying the value in the Quantity Removed text field.

Retiring Containers

When you retire a container, the container is not deleted, but moved to the Disposed location. Containers are moved to the Disposed Location when they are no longer in use. For example, you can retire a container if the substance stored in it has expired. When a container is retired, its status automatically changes to disposed. To confirm you have appropriate privileges to retire containers, see Roles and Privileges.

To retire a container:

  1. Navigate to the location of the container you want to retire in the Location Tree.
  2. Select the container in the Container List. The details of the container display in the Container Details.
  3. From the Container Details, Update menu, select Retire Container.

The Retire an Inventory Container window appears.

Note: For information about the fields in the Retire an Inventory Container dialog, see Inventory Enterprise Form Fields.

By default, both the Container Status drop-down list and Destination Location text box are set to Disposed.

  1. Click OK to retire the container.

Splitting Containers

When you split a container, all contents of the original container are transferred to new containers and the original container is discarded. To confirm you have the appropriate privileges to split a container, see Roles and Privileges.

To split a container:

  1. Navigate to the location of the container you want to split in the Location Tree.
  2. Select the container in the Container List. The details of the container display in the Container Details.
  3. From the Container Details, Create menu, select Split Container.

The Create a Set of Containers from the Original Container window appears.

  1. Barcode Description: Select container barcodes from the drop-down list.
  2. Sample Location ID: Enter the Location ID in the text field where you want to store the new containers.
  3. Sample Container Type: Select the container type from the drop-down list.
  4. Number of Containers: Enter the number of new containers in the text field that you want to create from the original container.
  5. Container Size (mg): Specify the size of the new containers in the text field.

Note: The Original Quantity (mg) field is not-editable. The value in this field pertains to the original container.

  1. Click Next. The Create a Set of Containers from the Original Container window appears.

  1. Container1 Quantity (mg): Enter the amount of the contents to be transferred to each new container, in the respective text field(s).

Note: When specifying the container quantity for new containers, ensure the sum of the quantities of the new containers is exactly equal to the quantity of the content of the original container.

  1. Click OK. The new containers with the specified quantity and size are created at the location you designate.

Requesting Samples

The Request Sample feature lets you request an aliquot from a container, or batch of containers.

Note: This feature can only be used when Grouping Batching Fields are set.

If the container you are requesting samples from does not contain enough quantity, the request is filled from other containers within the same batch. However, the request is not fulfilled completely if other containers in the same batch also do not contain enough quantity, or such containers do not exist. For example, if you request two 1-liter samples from a container containing only 1-liter of substance, and no other container in the same batch exists, then only a single 1-liter sample is created.

Note: If you cannot find the Request Sample menu item within the Request menu in the Container Details frame, see your system administrator.

Note: To request a sample after performing a batch search, click the Request link in the Batch search results and follow steps 4 through 9 in the procedure immediately below.

To request samples from a container:

  1. Navigate to the location of the container you want to request samples from in the Location Tree.
  2. Select the container in the Container List. The details of the container display in the Container Details.
  3. From the Container Details, Obtain menu, select Request. The Request Samples from an Inventory Container window appears.
  4. Delivery Location: Click Browse and select the delivery location.
  5. Date Required: Specify the date you want this sample delivered.
  6. Requested For: Specify the user for whom the sample is requested.
  7. Amount: Type the quantity of sample required.
  8. Fill in the other properties, if required.
  1. Click OK. A Request ID automatically generates.
  2. Click OK. A Requests tab is added to the Container Details.

The requests tab provides details about the sample, and you can request, edit, or cancel the sample request.

Lineage

The Lineage link lets you determine the complete hierarchy of a container. When you select a container in the Container List frame and click Lineage, a tree structure appears.

The tree structure contains the parent and child containers of the selected container, if any. It also contains all other containers with the same parent container as the selected container. In addition, the tree structure lets you determine if the parent container (of the selected container) has a parent.

The Update Containers link is available in the Update menu of the Container Details frame, but only when the Container List frame is in the multi-select mode. To view the Container List frame in the multi-select mode, you need to click the Multi-Select link in the Container List frame.

The Update Containers link lets you update multiple containers, at once.

To update multiple containers using the Update Containers link:

  1. Navigate to the location of the containers you want to update in the Location Tree.
  2. From the Container list, View menu, select Multi-Select.
  3. Check each checkbox to select the containers that you want to update.

  1. From the Container Details, Update menu, select Update Containers. The Update Inventory Container window appears.

  1. Select the field(s) you want to update in the Container Fields list.

Note:You can select multiple fields in the Container Fields list box. To select multiple fields, hold down the <CTRL> key while you make your selections.

  1. Click OK. The Update Inventory Container window appears, where you to enter values for the fields you selected in step 5 above.

Note: The window displays only the fields you have selected to update.

  1. Enter the values you want to update in the respective fields.
  2. Click OK.
