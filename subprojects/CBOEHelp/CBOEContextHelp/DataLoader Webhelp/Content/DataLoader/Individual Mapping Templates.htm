Individual Mapping Templates

Although the prospect of editing xml can be a bit scary, it can actually be as simple as a copy-and-paste operation with a very small amount of editing. Users may undertake this task themselves using these templates.

When creating your first data-mapping file, start with an empty text file using Windows Notepad. The first step is to copy the mapping-container template.

Listing 1: Mapping-container template

<mappings>
<destinationRecordType>RegistryRecord</destinationRecordType>
<!-- put all mapping templates between here...-->
<!-- ...and here, one after another. -->
</mappings>

Once you have established how your data-file’s fields will be used, you copy the necessary template for each data-mapping into this file, starting at the location specified. The only things you need to edit are:

  1. Green highlighted text- value must match your Registration System’s configuration
  2. Pink highlighted text - value must match a field in your data-file

Compound custom property

<mapping>
<enabled>true</enabled> <objectBindingPath>this.ComponentList[0].Compound.PropertyList['PROPERTY_NAME'].Value</objectBindingPath>
<memberInformation memberType="property">
<type>instance</type>
<description>Component custom property</description>
<args>
<arg index="0" input="derived" type="string">
<value> field_name</value> </arg>
</args>
</memberInformation>
</mapping>

Compound identifier

<mapping>
<enabled>true</enabled>
<objectBindingPath>this.ComponentList[0].Compound</objectBindingPath>
<memberInformation memberType="method">
<type>instance</type>
<name>AddIdentifier</name>
<description>new identifier</description>
<args>
<arg index="0" input="constant" type="string">
<value>identifier name</value>
</arg>
<arg index="1" input="derived" type="string">
<value>field_name</value>
</arg>
</args>
</memberInformation>
</mapping>

Compound’s structure

For SDFile format, the pink text must be: structure.

For ChemFinder.Net, the pink text must be: structure.

All other file types: use the name of the field containing the structure.

<mapping>
<enabled>true</enabled>
<objectBindingPath>this.ComponentList[0].Compound.BaseFragment.Structure.Value</objectBindingPath>
<memberInformation memberType="property">
<type>instance</type>
<name></name>
<description>structural representation</description>
<args>
<arg index="0" input="derived" type="string"> <value>structure_field_name</value>
</arg>
</args>
</memberInformation>
</mapping>

Compound’s default structure

Use this mapping for scenarios when the data-file lacks a structure field altogether. Also, if the file has a structure field that is sometimes empty, the user can place this snippet immediately after the “Compound’s structure” snippet to provide a default structure to prevent the data-record being rejected as invalid.

The text highlighted in green can be a -1, a -2 or a -3, corresponding to the three ‘no-structure’ options in the Registration application.

<mapping>
<enabled>true</enabled>
<objectBindingPath>this</objectBindingPath>
<memberInformation memberType="method">
<type>instance</type>
<name>AssignDefaultStructureById</name>
<description>Provide a default structure if none is been mapped</description>
<args>
<arg index="0" type="int" input="constant">
<value>0</value>
</arg>
<arg index="1" type="int" input="constant">
<!-- the default should be the 'No Structure' placeholder --> <value>-2</value>
</arg>
</args>
</memberInformation>
</mapping>
