Enabling the Pre-Load Record Matching

To enable pre-load record-matching from either version of the Registration data-loading applications, new configuration settings are required. These can be modified via the ‘Extended Duplicate-checking’ tab from Registration’s System Settings page. By default, the chemical structure is used as the preferred means to determine if a compound matches an existing, registered compound. However, when no structure is available (or one of the non-structure placeholders is applied via Registration user interface), matches will be screened instead by either:

  • A custom property’s value (for properties configured at the compound level), or
  • An identifier type’s value (for identifiers configured at the compound level)

If the ‘Type’ is set to Property or Identifier, the ‘Value’ must match an existing custom field (property) or Identifier name. If the ‘Value’ field is left empty, then the structure is used as the default matching mechanism. Structure-matching is configured in the ‘Duplicate Checking’ tab.
