Accessing the New Data Loader

The new Data Loader is launched via a separate executable.

To launch Data Loader:

  1. Log in to CBOE server
  2. Navigate to: C:\Program Files (x86)\PerkinElmer\ChemOfficeEnterprise\DataLoader2
  3. Double-click on "DataLoaderGUI.exe", the "Select Imported File" dialog displays

Note: If you are launching the DataLoader on your client, before running the "DataLoaderGUI.exe" you need to perform the following:

  • Copy the DataLoader2 folder into your client
  • Open the "DataLoaderGUI.exe.config" in edit mode, replace the localhost with the server name and save

<appSettings>
<add key="ApplicationName" value="DataLoader GUI"/>
<add key="AppName" value="REGISTRATION"/>
<add key="ChunkSize" value="10"/>
<add key="CslaAuthentication" value="Csla"/>
<add key="UseThreading" value="True"/>
<add key="NumberOfThreads" value="6"/>
<add key="EnableTrace" value="true"/>
<add key="ShowStackTrace" value="true"/>
<add key="CslaDataPortalProxy" value="Csla.DataPortalClient.WebServicesProxy, Csla, Version=2.1.1.0, Culture=neutral, PublicKeyToken=93be5fdc093e4c30"/>
<add key="CslaDataPortalUrl" value="http://localhost/COEWebServiceHost/WebServicePortal.asmx"/>
<add key="ClientSettingsProvider.ServiceUri" value="" />
</appSettings>

Multithreading

You can set the "UseThreading" attribute to True to enable the multithreading feature.

The multithreading feature is used to generate multiple threads at the same time and than to load data in parallel. The number of threads should be an integer, not null, not empty and preferably less than 10.

Example:

If you have a file with 100 records, UseThreading=True, NumberOf Threads=5 and ChunkSize=10.

The number of threads/processes created will be 5 and for each process 10 records are loaded together:

Process 1 starts from record 1 to 20
Process 2 starts from record 21 to 40
Process 3 starts from record 41 to 60
Process 4 starts from record 61 to 80
Process 5 starts from record 81 to 100

  1. Browse to the Input File (Three types of files are supported:.sdf, .xls and .txt). You can specify a range of the records to be imported by entering the start/end records in their corresponding fields, this is mainly used when your input file is large and contains an important number of records.
    Note: when you are importing an excel file, you need to select the worksheet and the header and specify the delimiter.
  2. (Optional) Click on the Split File button to split the Input file into smaller files. The number of the new files that will get created is based on the number of records you specify in the "SplitForm"dialog that appears on the button click as shown in the screen shot below.
    Example: If the Input File contains 3300 records and you enter 1000 in the SplitForm dialog, then 4 new files will be created.

  1. (Optional) The following dialog displays, listing the new files created

  1. Enter your credentials under the Login section and click on Next button
  2. The Field Mapping dialog displays allowing you to map the output fields with the input fields. The input fields from the source data file are mapped to the fields in the database where the data is to be loaded. For more details please refer to Output Fields

Note: When Row Level Security is set at the Batch Level, then Projects field appears in the Batch Information section. When the Row Level Security is set at the Registry level or is OFF, then the Projects field displays at the Registry Information section.

Note: for .xls, .csv and .txt input files, the structure column header should be available as "structure". The .xls and .csv files support the smiles for structures format.

  1. (Optional) Click on Save Configuration button to save a mapping template for future use. The Save Settings dialog displays, enter a name for the template in the Save to text box and click on OK button

  1. (Optional) In the Field Mapping dialog, click on Load Configuration button to load a previously saved template. The Load Settings dialog displays, select the template to be loaded and click on OK button.
    To rename a template; select the template in the list, click on rename button and enter the new name.
    To delete a template, select the template in the list, click on Remove button.

  1. After completing the mapping fields, click on Next button the following dialog displays:

  1. Before importing the records, click on the scan button to check for unique, invalid or duplicate records in the file. After the scan is complete, the results are displayed in the scan progress dialog and populated automatically in the "Import and Duplicate Check" dialog

  1. The previous dialog has two main sections: "Duplicate Check" and "Import". In the Duplicate Check section, the total number of the records in the input file and scan results are displayed. In the Import section, there are multiple import options that you can select as desired:
    • Submit for review: will submit the records into the temporary table
    • The other import options lets you load substances into both temporary table and permanent table of Registration Enterprise. Depending on the circumstances and Duplicate Action settings, some substances will go into temporary table instead of permanent table.
  2. Depending on the selection you make, specific or all records will be imported:
    • All records: the total records will be imported
    • Unique records: only unique records will be imported
    • Duplicate records: only duplicate records will be imported
    • Selected records: only selected records will be imported
    • Note that it is not possible to import invalid records
  3. To select records, check the checkbox corresponding to one of the options listed in point 13 above and click on View button, the following view form displays

  1. In the ViewForm dialog, you can select the records manually and one by one by clicking on the corresponding checkboxes or you can click on Select All button to select all the records at once. When you have a large number of records, you can click on Batch Mark button and specify the range of the records to select

    Click on "Inverse" button, to inverse the selection by selecting the unselected records and unselecting the selected records.
    After selecting a number of records, click on "Use Selected" button to import the selected records to the temporary or permanent database or click on "Export to" button to export the selected records and save it locally for future use.
    Click on Cancel button to go back to the "Import and Duplicate Check" dialog.
  2. After selecting the import option and the records to be imported, click on Import button. The Import progress dialog displays showing a report for the imported records. To view the report in details click on "Copy Log to ClipBoard" button and open the log file for review.
    Note that clicking on "Copy Log to ClipBoard" button will copy the log file path. In order to display the file, go to Start>>Run, paste the path and press Enter on your keyboard. The file is opened.

From the "Import and Duplicate Check" dialog, you can export records at any time and save them for later use. From the "Import and Duplicate Check" dialog, you can save the mapping file for later use by clicking on "Save Mapping File" button and you can browse to a new input file for import by clicking on "Import Another File" button.
