Testing the COERegistrationServices SOAP methods through the asmx page

Standard "CRUD" methods provided:

  temporary permanent
Create CreateTemporaryRegistryRecord CreateRegistryRecord
Retrieve RetrieveTemporaryRegistryRecord RetrieveRegistryRecord
Update UpdateTemporaryRegistryRecord UpdateRegistryRecord
Delete DeleteTemporaryRegistryRecord  

Also certain "execute" methods:

temporary permanent
GetTempID CheckUniqueRegistryRecord
  RetrieveMatchingFragmentList
  SaveRegistryRecord

If your Registration database is empty you will want to use the Registration system to create at least one temporary and one permanent registry record.
Note the integer ID of the temporary record; probably 1; and note string RegNum of the permanent record; probably "AB-000001".

You can test RetrieveTemporaryRegistryRecord using the ID of a known record.
Having retrieved an existing record you can copy the MultiCompoundRegistryRecord XML to notepad for use in further tests.
Now you can use GetTempID to get a new unused ID. Note the ID.
If you attempt to RetrieveTemporaryRegistryRecord using the new ID you will get an "Unable to GetRegistryRecord" error.
Now modify the XML from the earlier retrieve by changing the ID tag to the new ID value.
Invoke CreateTemporaryRegistryRecord and submit the modified XML to create a new temporary registration record.
The response will match the new ID value.
You can also use RetrieveTemporaryRegistryRecord to verify that the new record has been added.
You can modify values in the XML and submit the modified XML to UpdateTemporaryRegistryRecord and use RetrieveTemporaryRegistryRecord to verify that the update took place.
[Not all fields are updateable and some tags require that the update="yes" attribute be added?]
Finally, DeleteTemporaryRegistryRecord can be used to delete a temporary registry record.
The deletion can be confirmed by trying to use DeleteTemporaryRegistryRecord again or by attempting to RetrieveTemporaryRegistryRecord.

You can test RetrieveRegistryRecord using the RegNum of a known record.
Having retrieved an existing record you can copy the MultiCompoundRegistryRecord XML to notepad for use in further tests.
You can submit this XML unmodified to CreateRegistryRecord using a duplicateAction of D to create a duplicate registry record. The response will be the new RegNum.
You can exercise the other  duplicateAction values as well.
You can modify values in the XML and submit the modified XML to UpdateRegistryRecord and use RetrieveRegistryRecord to verify that the update took place.
[Not all fields are updateable and some tags require that the update="yes" attribute be added?]
You can pass the XML into CheckUniqueRegistryRecord and the response will consist of the duplicates, if any. Generally the Z should be C although M is available.
If the record would be unique they response is an empty string.

RetrieveMatchingFragmentList retrieves a fragment list based on the provided searchExpression.
The searchExpression is comprised of one or more search terms separated by the | character.
For example the simple search term molweight < 100 would return all fragments with a MolWeight less than 100.
Naturally, you must have fragments in your database in order for fragments to be returned.
Other search terms you might try include molweight between 90 and 100.
Search expressions are more completely documented on the asmx page itself.

The SaveRegistryRecord method can be executed but without a corresponding retrieve it's hard to test.
