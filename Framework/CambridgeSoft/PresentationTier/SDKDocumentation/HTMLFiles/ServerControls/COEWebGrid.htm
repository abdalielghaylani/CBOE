COEWebGrid Control

 

The COEFramework COEWebGrid server control provides a convenient way to render the results of database search in a rich hierarchical tabular representation. The main table shown in the grid is known as base table or parent table and the tables that are shown in the cells of the parent grid are the child tables. The various features of COEWebGrid control are:

 

  • Implements one-to-many relationship to the base table. 
  • Enables mark/unmark the individual records.
  • Enables mark/unmark all records with a single click.
  • Enables display the chemical structures in any field by embedding the ChemDraw Plugin Images.
  • Enables independently expand/collapse the child tables.
  • Enables expand/collapse all the child tables within a column.
  • Enables sort the base table rows any visible base field value.
  • Enables sort the child table rows by any visible child field.
  • Enables fix the table header, which helps table header to remain visible when rows are scrolled down.
  • Enables fix the first column to make it visible when columns are scrolled horizontally.

 

 

  • Supports grand child tables.
  • Enables export the results to Excel, BioViz (custom xml format), and sdf (industry standard delimited text format).
  • Enables modify the fields and field names to be exported to sdf using export templates.

 

 

Working with COEWebGrid

 

You can simply drag the COEWebGrid control from the Toolbox to your Web Page. The user can bind externally with the dataset or by providing the XML input files, DataView, ResultsCriteria, and SearchCriteria.

 

To use the COEWebGrid control:

 

1.       Create a Visual Studio Web Application

2.       Add the COEFramework server controls to the Toolbox.

3.       Drag the COEWebGrid control from the Toolbox to the form.

4.       Edit the following properties of the COEWebGrid control according to your requirements.

·         ID: Specify the programmatic name of the control.

·         BindExternally: Need to set to false to specify the XML files as the input for the COEWebGrid.  To externally bind the COEWebGrid, you need to specify the value of the property to true.

·         DataViewXml: Specify the XML as a string for the DataView.

·         DetailViewURL: Specify the URL for the hyperlinked column in the COEWebGrid.

·         ResultCriteriaxml: Specify the XML as a string for the ResultsCriteria.

·         SearchCriteriaXml: Specify the XML as a string for the SearchCriteria.

·         ShowCheckBoxColumn: Specify whether you want to display the CheckBox column in COEWebGrid.

·         CollapseImage: Specifies the name of the collapse image, clicking on to which collapses the child table/tables.

·         ExpandChild: Enables you to specify whether the child grids in the COEWebGrid should be expanded or collapsed when the COEWebgrid is loaded on the Web page.

·         ExpnadImage: Enables you to specify the name of the expand image, clicking on to which expands the child table/tables.

·         FontSizeInPoint: Enables you to specify the font size of the text in the column header.

·         ImageFolder: Enables you to specify the path of the image folder where images need to be saved.

·         DetailViewURL: Specify the URL for the hyperlinked column in the COEWebGrid.      

·         TargetDetailView: Specify the Target Frame for the DetailsViewURL.

·         NavigationImage: Enables you to specify the path of the hyperlinked navigation image.

·         NavigationImageHeight: Specifies the height of the navigation image.

·         NavigationImageWidth: Specifies the width of the navigation image.
