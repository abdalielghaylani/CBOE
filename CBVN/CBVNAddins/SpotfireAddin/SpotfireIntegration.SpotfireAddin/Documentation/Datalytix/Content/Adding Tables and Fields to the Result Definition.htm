Adding Tables and Fields to the Result Definition

Tables and fields are added to the result definition by:
• Drag and drop any table or field from the navigation tree.
• Right mouse click on any table/field from the navigation tree.
• Double click on field in the navigation tree.
When adding a table to the result definition, you can select to include:
• Default Fields
• All Fields
• Unique Key and Structure
These options are available for selection from the ‘When adding a table include’ drop down list located below the Available Tables and Fields pane. Alternatively, when adding a table, you can right mouse click on the table name to see these same options.
Note: When a grandchild table is added to the result definition tab, its parent table (child table) is also added.
 

To add tables to the result definition

Drag and Drop

1. Select a table on the navigation tree.
2. From the When adding a table include drop down list, select to include Default Fields, All Fields, or Unique Key and Structure.
3. Drag the table onto the result definition form.
4. Wait until the red line appears, The Red line indicates the position of the drop.
 

 
5. Drop the table onto the result definition form at the desired position.
 

 
Right Context Menu
1. Right mouse click on the table in the navigation tree.
2. Select Default Fields, All Fields, or Unique Key and Structure to identify what fields to include.
The table will be appended to the bottom of the query form.
 

To add fields to the result definition

Note: When a field is added to the table more than once, an alias field name is created. The field name is appended with a sequential number scheme.
Drag and Drop
1. Select a field on the navigation tree. Drag the field onto the result definition form.
2. Wait until the red line appears. The Red line indicates the position of the drop.
 

 
3. Drop the field onto the result definition form at the desired position.
 

 
Right Context Menu
1. Right mouse click on the field in the navigation tree.
2. Select Add.
The field will be appended to the bottom of the result definition form for the respective table.
Note: If the table to which the field belongs is not already part of the form, it will be appended to the form along with the field.
 
Double clicking on a field in the navigation tree
1. Double click on the field in the navigation tree.
The field will be appended to the bottom of the form for the respective table.
Note: If the table to which the field belongs is not already part of the form, it will be appended to the form along with the field.
The result criteria must contain at least one field.

Note: The order in which the fields within a table are displayed on the Result Definition form is the order in which they will be displayed in the Spotfire table. The fields can also be sorted in an ascending or descending order. Refer to the following section on Sorting the Results.

To remove tables/fields from the result definition

1. Drag and drop any field or table from the Result Definition tab to the left Tree panel. The table/field will be removed with the form and returned to the navigation tree.
OR
2. Right mouse click on the table or field in the form and select Remove Table or Remove Field respectively. The table/field will be returned to the navigation tree.
Note: The base table cannot be removed from the Result Definition form.
